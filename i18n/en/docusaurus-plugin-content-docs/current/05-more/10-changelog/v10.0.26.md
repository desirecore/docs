# DesireCore v10.0.26

**Release Date**: 2026-04-09

![v10.0.26 更新概览](/img/changelog/v10.0.26-en.png)

## New Features
- **Built-in Global Skill Extensions**: Added 7 built-in global skills, including skill-creator, Word, PDF, Excel, PPT, web access, and frontend design. When you enter related requests, the agent can more naturally and automatically invoke the corresponding capabilities to handle document processing, online search, page design, and skill creation.
- **Exploratory Delegation Preset**: The agent can now launch read-only "Explore" exploratory subtasks, which are better suited for file search, code discovery, and content analysis in large projects.
- **Dedicated SendUserMessage Message Cards**: Notification messages proactively sent to you by the agent are now displayed as standalone cards, with support for Markdown, attachments, and proactive notification tags for a more intuitive reading experience.
- **Runtime Environment Panel**: A new runtime environment view has been added to the Explorer, where you can check details of your local runtime environment and available tools, providing clearer environment information for using related capabilities.

## Fixes
- Fixed an issue where Token usage statistics could be overstated in some cases. Historical activity records, in-conversation statistics, and summary data are now more accurate.
- Fixed an issue with how Token usage was displayed for models with cache-based billing. Cache read/write usage is now shown separately and is no longer double-counted with regular input.
- Fixed an issue where activity record usage would be lost when clearing a conversation. Real usage records are now retained for audit viewing even after chat content is cleared.
- Fixed an issue where SendUserMessage messages would only appear after generation was complete. Message cards now appear immediately and update progressively as content streams in.
- Fixed an issue where SendUserMessage cards were rendered incompletely in some historical messages and merged views. Display is now more stable when reviewing conversations.

## Improvements
- **All-New Brand Visual Identity**: The app has been updated with a brand-new visual identity and icon assets, creating a more unified overall interface.
- The default model options in agent configuration have been consolidated into a clearer configuration structure, making imports and compatibility with older configurations smoother.
- Usage information for the current query in conversations has been changed to a clearer itemized display, so you can more intuitively see the breakdown of input, output, cache, and cost.
