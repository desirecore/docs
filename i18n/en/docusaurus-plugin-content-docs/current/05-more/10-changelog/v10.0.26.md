# DesireCore v10.0.26

**Release Date**: 2026-04-09

![v10.0.26 更新概览](/img/changelog/v10.0.26-en.png)

## New Features
- **Built-in Global Skill Expansion**: Added built-in skills such as skill-creator, docx, pdf, xlsx, pptx, web-access, and frontend-design. Users can now create skills more naturally, process office documents, access web content, and generate frontend design solutions.
- **Explore Delegation Preset**: Agents now support delegating read-only exploratory subtasks in Explore mode, making it better suited for file searching, code lookup, and content analysis.
- **Dedicated SendUserMessage Message Cards**: Messages proactively sent by the Agent to the user are now displayed as standalone cards, with support for Markdown, attachments, and proactive notification flags for clearer reading.
- **Real-Time Streaming Display for SendUserMessage**: When the Agent is composing a message to send to you, the message card now appears immediately and updates progressively in real time, without waiting for the full message to be generated.

## Fixes
- Fixed an issue where token usage statistics were overestimated in some scenarios. Usage data in activity logs and query summaries is now more accurate.
- Fixed an issue where historical cumulative token credentials distorted statistics for old conversations. After the upgrade, old data will also be automatically corrected in a more reasonable way.
- Fixed an issue where clearing a conversation also caused token data in activity logs to be lost. Token consumption is now preserved for auditing even after chat content is deleted.
- Fixed an issue where cleared old sessions could be reused by the same Agent. New conversations will now correctly start from a new session.

## Improvements
- **Upgraded Token Display**: Query summaries and the runtime status panel now distinguish more clearly between input, output, cache reads, cache writes, and cost, making it easier to understand the actual consumption of each query.
- **Brand Visual Refresh**: The app icon and parts of the interface visuals have been updated with a new brand style for a more consistent overall identity.
- **Improved Compatibility for Agent Model Configuration**: The default configuration structure for Agent models has been optimized, while maintaining compatibility when importing and reading old configurations, reducing the impact of configuration migration.
