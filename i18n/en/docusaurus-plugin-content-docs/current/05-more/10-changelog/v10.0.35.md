# DesireCore v10.0.35

**Release Date**: 2026-04-20

![v10.0.35 更新概览](/img/changelog/v10.0.35-en.png)

## New Features
- **Built-in Windows PowerShell Tool**: On Windows, the agent can now directly invoke PowerShell to run commands and scripts, with support for approvals, timeout control, background execution, and large-output handling.
- **Continue Generating Button in the Input Box**: When a response is interrupted or paused, you can now resume generation with one click directly next to the input box, without relying solely on the entry point in the message bubble.

## Fixes
- Fixed issues on Windows where the PowerShell tool produced no output, Chinese text was garbled, approval was mistakenly triggered for read-only commands, and prompts were inaccurate across different PowerShell versions.
- Fixed an issue where multilingual characters and emoji in the output of built-in tools such as PowerShell and Bash could be mistakenly removed or truncated, making command results more complete and reliable.
- Fixed an issue where the unread red dot would sometimes not disappear immediately after switching to a conversation.
- Fixed an issue where the agent status could still incorrectly display as "Working" after generation was paused or the conversation was cleared.
- Fixed an issue in some Windows NAT port-occupation scenarios where the Mail Center and Workflow services could not connect properly, requests failed, or failures occurred silently.
- Fixed an issue where some long conversations were not compressed automatically in time when the context was nearing its limit, reducing the risk of overflow caused by inaccurate context estimation.
- Fixed issues where, after model preset merging, deprecated models still appeared in the chat model list and model ordering became incorrect.

## Improvements
- **Streaming Conversation Performance Optimization**: The main chat area and conversation list are now smoother during streaming output, reducing lag and unnecessary refreshes caused by message changes in other conversations.
- Further optimized the model list synchronization logic, making the default order of preset models more accurate and no longer showing deprecated models in chat scenarios.
- Zhipu GLM and related integrations can now display the reasoning process more reliably after reasoning mode is enabled.
- The internal test model list has been updated, adding a new high-speed default model and restoring MiniMax-M2.5 as an alternative model.
