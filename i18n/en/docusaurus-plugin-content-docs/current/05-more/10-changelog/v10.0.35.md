# DesireCore v10.0.35

**Release Date**: 2026-04-21

![v10.0.35 更新概览](/img/changelog/v10.0.35-en.png)

## New Features
- **Built-in Windows PowerShell Tool**: Windows users can now let the agent use PowerShell directly to execute commands and scripts, with support for approval, timeout control, background execution, and large output handling.
- **Continue Generating from the Input Box**: When a reply is interrupted and it's inconvenient to operate from the message bubble, you can now continue generation with one click right next to the input box, making it easier to complete the conversation.

## Fixes
- Fixed issues with the PowerShell tool on Windows where there was no output, Chinese text was garbled, approvals were incorrectly triggered for read-only commands, and syntax hints were inaccurate across different versions.
- Fixed an issue in built-in command output handling where emoji and multilingual characters could be mistakenly removed. Chinese, Japanese, Korean, Arabic, Hebrew, and emoji are now preserved correctly.
- Fixed an issue where the chat header would still incorrectly show a lingering "Working" status after generation was paused or the conversation was cleared.
- Fixed an issue where the unread red dot in the sidebar would not disappear promptly after switching to a conversation with existing unread messages.
- Fixed an issue in some Windows environments where the Mail Center could not connect due to dynamic server port changes, causing data loading failures or repeated WebSocket connections.
- Fixed an issue in some Windows environments where the workflow service could become inaccessible due to port conflicts or startup timing issues, or requests could be sent to the wrong port.
- Fixed an issue where some models still could not display their thinking process after Thinking Mode was enabled. Zhipu GLM and related integrations can now properly show thinking content.
- Fixed inaccurate conversation budget estimation in long-context scenarios, reducing cases where context limits were exceeded without timely automatic compression, which could affect replies.
- Fixed issues after model preset synchronization where old models remained, sorting was abnormal, and deprecated models still appeared in the chat model list.

## Improvements
- Interface refreshes during streaming conversation generation are now smoother, reducing interference from background conversation updates on the current chat view.
- Sidebar message previews now update only after a message is completed or newly added, resulting in more stable overall chat performance.
- The preview model list has been updated. The default model has been changed to a higher-priority high-speed version, and MiniMax-M2.5 is once again available as an alternative model.
