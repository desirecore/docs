# DesireCore v10.0.77

**Release Date**: 2026-06-15

![v10.0.77 更新概览](/img/changelog/v10.0.77-en.png)

## New Features
- **Conversation Export**: You can now export an entire conversation or selected messages as Markdown or PDF, with the option to include tool calls, reasoning, and system messages as needed.
- **ChatGPT Subscription Integration**: You can now directly reuse your local ChatGPT/Codex subscription credentials to access related models, without needing to enter an additional API key.
- **Billing and Balance Center**: You can now purchase compute credit packages in the client, and view your balance and upcoming credit expiration reminders in your profile, the sidebar, and the resources panel.
- **Team Popover and Settings Entry**: The team TL badge in the conversation list now supports expanding a member popover, allowing you to quickly view members, jump to the corresponding chat, and open team settings.
- **Minimum Version Upgrade Guidance**: When your current client version is lower than the official minimum supported version, the app will provide a clear upgrade prompt and guide you to check for updates.

## Fixes
- Fixed an issue where attachments from a user message would remain above the input box after clicking "Retry."
- Fixed an issue where the message bubble would suddenly become narrower and text would display incompletely when editing a user message.
- Fixed an issue where newly imported agents or teams would not immediately appear in the sidebar and would only show up after a manual refresh.
- Fixed an issue where cloning an agent could fail when only "Relationships" or "Root-Level Preferences" was selected.
- Fixed an issue where the primary controller agent could still be cloned, preventing accidental operations from affecting the system’s core role.
- Fixed an issue where some requests would fail due to incompatible role fields when using models such as DeepSeek through Compat Proxy.
- Fixed an issue where agents could occasionally fall into repeated output, flooding the screen, or silently fail after delegated execution.
- Fixed an issue in development environments where, after a restart, the frontend could occasionally fail to automatically reconnect to the real-time channel, causing approval cards and other features not to appear.
- Fixed an issue in "Apps & Services" where some official HTTP services incorrectly displayed an unavailable "Test Reachability" button and returned an internal error.
- Fixed an issue where "Clear Local Data" appeared to succeed in development mode or under a custom data directory but did not actually clear anything, and prevented the UI from getting stuck in a loading state when the cleanup process failed.
- Fixed a gap where installed apps could not derive services; related apps can now correctly generate usable service entries.
- Fixed an issue where skill names and descriptions in the skill list would not update to the localized language after switching the interface language.

## Improvements
- **PDF Reading Experience**: PDF reading now handles long documents and scanned files more intelligently on a page-by-page basis, supports continuing from where you left off, and directly uses vision models to understand scanned page content when available.
- **Apps & Services Catalog Experience**: You can now directly open derived service details, manually re-derive services, and see real-time updates to the number of services pending approval.
- **Linux Startup Installation**: The Linux AppImage version now includes more stable launcher installation support, making it easier to create command entries and desktop shortcuts.
