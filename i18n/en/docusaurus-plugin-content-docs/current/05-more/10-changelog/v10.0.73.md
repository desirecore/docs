# DesireCore v10.0.73

**Release Date**: 2026-06-05

![v10.0.73 更新概览](/img/changelog/v10.0.73-en.png)

## New Features
- **Offline Diagram Editing and Conversational Diagram Generation**: You can now use draw.io for professional diagram editing completely offline, and have the Agent directly generate editable diagram cards in chat, such as flowcharts and architecture diagrams.
- **Runtime Environment Injected into Agent Context**: The Agent can now detect the current operating system, available toolchains, and information about DesireCore’s built-in Node/Python runtimes, allowing it to generate and execute task plans more accurately.
- **Secure Authorization for External Service Credentials**: You can now securely authorize the Agent to access external capabilities such as HTTP, MCP, scripts, and Bash through placeholders, preventing plaintext credentials from being exposed to the Agent, model context, or logs.
- **Offline Office Dependencies Preinstalled**: Dependencies required for Office-related skills now come preinstalled with the client and are reused across platforms, making first-time use of these skills faster, available offline, and free from any extra environment setup.

## Fixes
- Fixed an issue where, in some cases, adding a model provider could corrupt `compute.json`, causing the model list to be cleared and forcing a reinstall. Configuration issues are now automatically validated, backed up, and restored whenever possible.
- Fixed an issue where Kimi could fail outright during multi-turn tool calls in thinking mode due to missing reasoning content fields.
- Fixed an issue where, after chat history was persisted, draw.io diagram cards could degrade into a generic “tool executed” message and fail to re-render correctly after restart.

## Improvements
- The fixed connection for the web-access browser can now recover automatically if the target browser is closed or disconnected, allowing available browsers to be rediscovered without restarting the agent.
- After manually refreshing the runtime environment page, the system now clearly indicates whether the refresh succeeded or failed, preventing confusion about whether the action took effect.
- This release also syncs multiple global skill versions in the built-in skill marketplace, so users who have already upgraded can still receive these skill content updates normally.
- Following an upgrade to the application’s underlying runtime, overall stability and startup performance have improved.
