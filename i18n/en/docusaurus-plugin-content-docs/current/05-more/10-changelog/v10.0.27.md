# DesireCore v10.0.27

**Release Date**: 2026-04-10

![v10.0.27 更新概览](/img/changelog/v10.0.27-en.png)

## New Features
- **Global built-in email skills**: All Agents now have email capabilities by default, allowing them to directly handle sending and receiving, searching, drafts, label management, attachment downloads, and automation rules for Gmail, Outlook, and IMAP mailboxes.
- **Inline review of editor changes**: When an Agent modifies a document, you can now see additions and deletions directly in the editor and accept or reject them paragraph by paragraph, without switching to a separate diff view.
- **Selective data import and export**: You can now freely choose what to back up or restore by category, including Agents, skills, conversation history, system settings, media, email, and workspace files, and view data size and conflict warnings before importing.
- **Runtime environment now supports container detection**: The Runtime Environment page now detects and displays Docker and Podman, making it easier to check whether container-related components are available on your machine.

## Fixes
- When the model service returns an empty response or an invalid format error, conversations now display readable error messages more reliably instead of showing no indication or exposing raw JSON directly.
- Fixed an issue where newly imported Agents did not appear in the member list when creating a team; they can now be selected properly whether online or offline.
- Fixed an issue on Windows where data import could fail because of Chinese-language paths; backup packages containing Chinese directory names can now be imported normally.
- Fixed an issue where relative paths were resolved incorrectly when clicking document links in chat; files can now be opened more reliably from the corresponding Agent’s working directory, with clear prompts shown if opening fails.
- Fixed an issue where duplicate messages could appear during paginated loading and message merging, making conversation history browsing more accurate.
- Fixed an issue where installed Node.js versions in the Runtime Environment might not display correctly; the installed version list is now properly detected and shown.
- Fixed an issue where the Runtime Environment version dialog could be cut off at the edge of the interface; the dialog position is now displayed more completely.

## Improvements
- Downloads for the Node.js and Python toolchains in the Runtime Environment are now more stable on domestic networks, with added multi-node proxy support and direct-connection fallback improving installation success rates.
- Operations such as install and delete on the Runtime Environment page now show more timely real-time progress output, making the overall experience smoother.
- Data export now automatically excludes sensitive or irrelevant content such as keys, device identifiers, and scheduling state, making backups safer and more complete.
- During data import and export, large runtime directories in the workspace, such as dependency packages and virtual environments, are now skipped automatically to reduce backup size and lower the risk of import failures.
- When you ask an Agent to write, edit files, or generate documents, the system now identifies your intent more accurately and optimizes prompts accordingly, improving the effectiveness of content generation and file operations.
- PDF-related capabilities have been upgraded, with further improvements to document viewing, text selection, font compatibility, and overall rendering experience.
