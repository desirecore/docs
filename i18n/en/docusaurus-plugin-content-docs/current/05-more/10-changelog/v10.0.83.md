# DesireCore v10.0.83

**Release Date**: 2026-07-12

![v10.0.83 更新概览](/img/changelog/v10.0.83-en.png)

## New Features
- **Claude subscription integration**: Supports three ways to access Claude Pro/Max subscription compute: native login via Claude Code, in-app OAuth, or manual `setup-token`. Fulfillment is handled through the built-in Claude Agent SDK backend with a complete audit chain.
- **Three-layer prompt system**: Introduces global (user-level), team (Git-managed, distributed with the team), and Agent layers of overlay prompts. A centralized “Prompt Center” UI provides unified management with structured form editing and real-time preview.
- **Thinking depth control**: A session-level thinking depth selector button added to the input box, supporting six levels: auto/off/low/medium/high/maximum. Automatically compatible with both OpenAI and Anthropic formats; the setting can be persisted in the agent configuration.
- **Windows GUI automation**: The Windows installer bundles cua-driver 0.7.1. Agents can directly control native Windows applications (screenshot, click, input, read window state), with support for background windows that don’t steal focus—ready to use out of the box.
- **Offline runtime out-of-the-box**: The installer includes Python 3.13 and Node.js 24 LTS runtimes, auto-imported on first launch and fully usable offline. The version list also provides an offline fallback, eliminating infinite retries on an empty list.
- **Built-in Git deterministic dependency**: Git 2.53.0 is bundled with the application, so marketplace sync, Agent snapshots, and other features work without a system Git installation.
- **AgentFS Git full-tree snapshot**: The rollback mechanism has been upgraded from file-level copying to Git full-tree snapshots, supporting holistic restoration (including recreation after deletion) and naturally covering off-round and background writes.
- **Standalone deployment mode**: Agent Service can be deployed as an independent process on a server, separate from Electron, with secure remote access and one-customer-per-container delivery (includes instance lock, token authentication, SSRF protection, Caddy TLS, and other complete recipes).
- **Context intelligent management**: Context trimming now follows a cascading strategy: truncate tool output first → delete old turns → fold middle segments. Tool outputs that can be refetched are unloaded losslessly. The model’s real context window and output limit are learned automatically from limit-exceeded errors and persisted, enabling large outputs to be generated in one pass.
- **Compute billing and provider announcements**: The compute model list now shows a “Compute power provided by partners” notice at the top. Models are partitioned into text/image generation/video generation sections (generation models are marked “Chat unavailable”). Pricing is displayed for different billing modes such as per-token or per-use.
- **File “Open with” dropdown**: A new “Open with ▾” dropdown on message file entries supports in-app open, system default application, choose other application, and reveal in folder.
- **Video generation upgrade**: The video generation relay has been upgraded to an OpenAI Video API-compatible path, providing out-of-the-box HappyHorse video generation using official cloud compute, while also adapting to the Doubao Seedance 2.0 video model.
- **Migrate to new device**: A complete “Migrate to new device” export/import wizard has been added, supporting M1 identity replacement (identity follows the person) and M2 merge into the current identity (source data remapped to the local identity).
- **Migrate API keys**: Migration exports can optionally encrypt API keys (scrypt + AES-256-GCM). On import, you can decide per provider whether to replace or merge keys, avoiding reconfiguring all keys on the new device.
- **Plan discount display**: Plan card pricing now supports a strikethrough original price and discount badges like “Save X%” / “Limited free”.
- **Manual refresh service catalog**: A manual refresh button has been added to the “Apps & Services” page to instantly sync the registry directory.
- **Compute preset merge conflict protection**: When config-center preset updates are applied, user-modified provider fields (endpoint, label, format, etc.) are never overwritten. Conflicts are presented in a review panel for visual comparison and decision.
- **Provider-level version threshold**: Config-center can declare a minimum client version requirement for a provider. If the requirement is not met, a “Client update required” prompt is shown instead of the provider being silently unavailable.
- **Provider concurrency limiting**: You can configure a maximum number of concurrent requests and a queue limit per provider. Requests exceeding the limit wait in the queue instead of erroring out, and the frontend shows real-time queue status.
- **Queued queries can be cancelled**: Queries waiting in the queue can be stopped by clicking, immediately dequeuing them so they are not unconditionally scheduled for execution with side effects.
- **Compute preset configuration version number**: The header of the compute page now displays the current preset data version, making it easy to confirm sync status.
- **Claude SDK session resume**: The Claude subscription path supports session-resume to restore prompt cache, reducing quota consumption from repeated context. Images can also be passed directly to the model on the subscription path.
- **Subtask timeout no longer auto-aborts**: The default synchronization timeout for delegation, generating sub-agents, and sending messages has been relaxed to 30 minutes. On timeout, the sub-session is no longer automatically aborted; instead, the caller is informed and can choose to continue or actively terminate.
- **Relationship graph delivery edge**: The relationship graph now includes a “delivery” edge. When an agent successfully delivers a result, the graph correctly renders the delivery link.
- **Conversation PDF export client rendering**: Conversation PDF export now uses client-side rendering, removing the dependency on an Agent Service host.
- **Smart prompt for non-vision model on scanned documents**: When a non-vision model reads a pure scanned PDF, instead of clearing tools and causing infinite re-reading, the server now directly prompts “Please switch to a model that supports visual understanding” and ends the turn.
- **SSRF protection per-agent**: SSRF protection has been extended from a global switch to per-agent configuration, with changes taking effect immediately without restart. The stricter of the global and agent-level settings is applied.
- **Ls tool execution optimization**: Ls directory listing defaults to automatic approval without manual confirmation (read-only operation). On macOS, when accessing a directory protected by system permissions for the first time, a native authorization dialog automatically appears; when permission is denied, a clear system-settings guide is returned.
- **Marketplace list includes pointer skills**: ADR-038 pointer skills are fully available when browsing, searching, and viewing details in the marketplace. Version determination has been fixed (falls back to the marketplace registered version when no version field is present), and frontmatter validation now accepts comma-separated values.
- **Message area scrollbar widened**: The vertical scrollbar track in the chat message area has been widened to 10px for easier mouse dragging.
- **Import/clone agent immediately usable**: Agents imported or cloned by any means are online immediately after being written to disk; a client restart is no longer required.
- **Delegate prompt path rules**: When delegating tasks, agents now actively use absolute paths when referencing files or directories, preventing path resolution failures caused by different working directories on the receiving side.

## Fixes
- Fixed an issue where macOS builds were rejected by Apple notarization because the built-in Python runtime binary lacked a secure timestamp (macOS packages now fall back to online runtime installation on first launch).
- Fixed an issue on Windows where asar path separators caused SDK file verification to falsely report all misses.
- Fixed an issue where `cpSync` default behavior rewritten built-in Git symbolic links as absolute paths, causing macOS notarization failure and dangling links in Linux packages.
- Fixed an issue where uploading large files over slow links caused an endless 403 `RequestTimeTooSkewed` loop because transfers exceeded the SigV4 time window (files ≥64MB now use S3 multipart upload).
- Fixed an issue in CI where `detect` and `post-release` jobs were scheduled on macOS runners, causing `gh` CLI download failures and the entire build being skipped.
- Fixed multiple after-pack platform bugs in the v10.0.83 build that caused consecutive build failures.
- Fixed an issue where same-named models routed through proxies could be incorrectly sent to the wrong provider (precise routing for multiple providers with same-named models, also covering all internal LLM invocation paths).
- Fixed an issue where gateways that did not declare a dialect would not send the thinking depth activation parameter when a depth was selected (parameters are now injected fully based on the dialect).
- Fixed an infinite loop where non-vision models reading pure scanned PDFs had all tools cleared, causing degenerate re-reading.
- Fixed an issue where scheduled tasks would respond only once and then stop executing (maxTurns relaxed to 30, interim-ack detection added for continued running, failure fallback closed loop).
- Fixed an issue where scheduled task execution results were not fed back, rendering all backoff, alerting, and disabling mechanisms ineffective.
- Fixed an issue where relative time drifted in conversations (the model’s “now” was frozen at session start, causing “X minutes later” to be inaccurate).
- Fixed an issue where clicking `file://` links in agent replies caused errors on macOS or a gray screen freeze on Windows.
- Fixed an issue where data cleanup failed on Windows (Git read-only files and reserved device names caused `rmSync EPERM`).
- Fixed an issue where imported agents appeared as offline in the list (the imported path was not registered in the Registry after being written to disk).
- Fixed an issue where AgentFS file monitoring hot updates completely stopped working (`chokidar` mistakenly ignored the entire `~/.desirecore` hidden directory tree).
- Fixed an issue where fan-out parallel delegation skipped target validation, causing non-existent targets to be silently executed.
- Fixed an issue where delegation cards displayed “unknown” in fan-out mode.
- Fixed an issue where concurrent delegation, messaging, and assistance requests resulted in mismatched replies (a `correlationId` was added to the via-messaging path for precise matching).
- Fixed an issue where `send_message` asynchronous receipts could mismatch synchronous waiters across paths due to a missing `correlationId`.
- Fixed an issue where pressing Shift+Enter for a new line in the chat input box scrolled the new line out of view, making users think the line break had not worked.
- Fixed an issue where scrolling to the bottom of the chat message area and continuing to scroll caused the entire window to flicker (added `overscroll-contain` to block scroll chain bubbling).
- Fixed an issue where a horizontal scrollbar in the message area made long content clipped and invisible (switched to `overflow-wrap: anywhere` plus markdown internal horizontal scrolling).
- Fixed an issue where the message action bar overflowed and covered the timestamp in narrow bubbles.
- Fixed an issue where the selection circle was unresponsive in multi-select mode (especially in complex DOM structures).
- Fixed an issue where the agent list indicator incorrectly showed green during tool execution (indicator color now derived from session state and active tool name).
- Fixed an issue where requests were routed to a local provider when both cloud and local models had the same name.
- Fixed an issue where the message count in conversation-memory topic details displayed values in the billions (`messageIndex` was mistakenly passing a timestamp).
- Fixed an issue where floating-point relationship heat numbers broke the card layout (now displayed with a thousands separator and one decimal place).
- Fixed an issue on Windows where paths containing spaces caused all Git operations to crash.
- Fixed an issue where Git 8.3 short-path conversion spammed the log on every Git operation when PowerShell was unavailable.
- Fixed an issue on Windows where extracting the built-in Git tarball always failed because MSYS tar cannot handle drive letter paths.
- Fixed an issue where application cold-start taskboard probing produced a large amount of 404 noise in the console.
- Fixed numerous data-consistency race conditions in concurrent multi-agent delegation (directory collisions, member loss, half-written files, count drift).
- Fixed an issue where the dual-lock system for `agent.json` was mutually unaware of each other, causing concurrent write conflicts.
- Fixed an issue where the provider semaphore used the wrong key, making rate limiting ineffective.
- Fixed an issue where a provider’s fractional concurrency limit was rounded down to zero slots, and configuration read failures were not retried.
- Fixed an issue where corrupted compute configuration at agent-service startup prevented the entire service from starting.
- Fixed an issue where stopping an agent query caused a permanent global concurrency slot leak, causing all top-level queries to hang.
- Fixed the same slot leak occurring through the system sleep and conversation reset abort paths.
- Fixed an issue where data import/export rules were maintained in a fragmented way.
- Fixed an issue where malicious entries during zip import could write outside the intended directory.
- Fixed an issue where the Agent Service did not automatically recover after an import failure.
- Fixed an error in `agent.json` schema validation (added `author` and `default_enabled` field definitions).
- Fixed an issue where renaming an agent did not refresh the chat sidebar list.
- Fixed issues where the agent list showed duplicate teams and residual team entries remained after deleting an agent.
- Fixed an issue where the service catalog pending-review window crashed due to dirty data and did not update status in real time.
- Fixed an issue where docx seed self-checks always failed with a `SyntaxError` due to nested quotes.
- Fixed an issue where non-official Anthropic-compatible gateways (DeepSeek/GLM, etc.) in reasoning mode were missing the thinking response, causing a 400 error.
- Fixed an issue where the self-healing mechanism for `max_tokens` exceeding a gateway’s hard limit did not recognize Chinese error messages, preventing it from learning.
- Fixed multiple boundary accuracy issues in context trimming and token estimation.
- Fixed an issue where models like GLM mistakenly output tool parameters as an array, crashing the entire session.
- Fixed an issue where during the pipeline assembly phase (after acquiring a slot, before `sessionId` generation) query cancellation was unresponsive.
- Fixed an issue where provider semaphore queue waiters had no abort awareness, leading to wasted queuing delays.
- Fixed an issue where pointer entries missing `requiredClientVersion` caused the upgrade gate to always evaluate as false.
- Fixed an issue in standalone mode where some requests bypassed the unified authentication entry point, rendering authentication ineffective.
- Fixed an issue where browser native media elements in standalone remote access could not carry authentication headers, making images and videos invisible.

## Improvements
- The Windows installer now includes cua-driver, so users no longer need to install it manually for GUI automation.
- Windows Git paths are automatically converted to 8.3 short paths, reducing simple-git warning noise.
- The chat message area scrollbar hit area has been widened to 10px for easier dragging.
- Taskboard detail probing now uses non-404 semantics, eliminating console noise during startup and session switching.
- Marketplace repository source saving has been decoupled from full sync, fixing the issue where adding a repository left it stuck on “Saving…” for a long time and showed the repository twice.
- The “Apps & Services” page now includes a manual refresh button for convenient instant sync of the registry directory.
- The compute page header displays the current preset configuration version number.
- Plan cards support strikethrough original pricing and discount display.
- Plan pricing is now displayed by billing mode, such as per-token or per-use.
- Cloud models are partitioned into text/image generation/video generation sections, with generation models marked “Chat Unavailable”.
- The compute provider notice is now driven by API data, removing hardcoded partner names.
- Runtime directories have been unified to `~/.desirecore`, with automatic migration for existing users.
- DNS rebinding protection has been enhanced (DNS pinning check before outbound + undici connection locking).
- Permission and security hardening for email center, workflows, and other integration features.
- CI gating has been merged and optimized (Typecheck and Unit Tests combined into a single job with `tsbuildinfo` incremental caching).
- CI disabled the setup-node default npm cache to eliminate massive outbound traffic from self-hosted runners.
- Standalone smoke test triggering has been tightened to only core path changes like `lib/**`.
- Dependency upgrades: Electron 43.1.0, Node.js internal upgrade, fastify 5.10.0, tiptap suite 3.27.3, typebox 1.3.5, and others.
