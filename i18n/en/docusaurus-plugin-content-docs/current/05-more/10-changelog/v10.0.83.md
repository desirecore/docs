# DesireCore v10.0.83

**Release Date**: 2026-07-11

## New Features
- **Claude subscription integration**: Sign in with a Claude account to use Claude compute. Three authentication methods are provided: local login, in-app OAuth, and manual token entry. Model requests are fulfilled via the official SDK backend, with a complete audit trail.
- **Windows GUI automation**: The Windows client includes a built-in cua-driver, allowing agents to directly operate desktop applications (screenshot, click, type, read window state) out of the box with no installation required.
- **Thinking depth control**: A new thinking-level button in the chat input area lets you choose from auto/off/shallow/medium/deep/deepest levels. The setting is remembered per conversation independently, and is compatible with both OpenAI and Anthropic formats as well as major model providers.
- **Three-tier prompt system**: A new layered system prompt structure with global (user-level), team-level, and agent-level prompts. All can be edited and managed centrally in a prompt center, and team prompts are distributed with the team's git repository.
- **Migrate to new device**: A "Migrate to new device" export wizard has been added to the Settings page. When importing on a new machine, the migration package is automatically recognized, supporting identity replacement or merging into the current identity. API keys can be encrypted and exported for restoration on a new device.
- **Offline runtime installation**: The installer bundles Python 3.13 and Node.js 24 LTS; on first launch they are silently imported in the background, making the environment fully offline-capable. Version lists now fall back to cached data, so an empty list is no longer shown when the network is unavailable.
- **Preset compute configuration update notifications**: When preset configurations in config-center are updated, a notification is pushed allowing you to review diffs item by item and choose to adopt upstream changes or keep local settings. User-customized parameters will not be silently overwritten.
- **File open options**: Files in chat now support an "Open with" dropdown menu, allowing you to open them in-app, with the system default app, with another app, or reveal in folder.
- **Plan discount display**: Paid plan cards now show strikethrough original prices and discount labels; limited-time free plans show a "limited free" badge.
- **Transparent self-healing for context overflow**: When a model's context limit is exceeded, the true window limit is automatically learned and persisted, so the next retry immediately self-heals. Large output budgets are dynamically allocated in one pass, eliminating the need for multi-step continuation.
- **Provider concurrency control**: You can set a concurrency request limit for a single model provider. Requests exceeding the limit will queue and wait, solving issues where multiple sub-agents calling the same provider simultaneously cause rate-limit failures.
- **Automatic switch prompt for scanned documents with non-vision models**: When a non-vision model encounters a pure scanned PDF, it now immediately prompts the user to switch models instead of falling into an infinite retry loop.
- **Client-side PDF export for conversations**: PDF export of conversations has been switched to local client-side rendering, so it works correctly even in remote standalone deployments.
- **Preset version number on compute service page**: The header of the compute configuration page now displays the current preset data version, making it easier to troubleshoot configuration sync issues.
- **Manual refresh on apps & services page**: A manual refresh button has been added to the service directory page, allowing you to actively pull the latest service list.
- **Identity recognition in data export**: Data exports now include identity metadata, and when importing, the source identity is checked against the local identity, with a warning if they do not match.

## Fixes
- Fix race conditions during concurrent multi-agent delegation, including data inconsistency, directory conflicts, count drift, and incomplete file writes.
- Fix global concurrency slot leaks after aborting queries, which caused all agents to freeze after 3 consecutive aborts.
- Fix concurrency slot leaks also triggered by system sleep or session reset.
- Fix queued queries that could not be aborted, continuing to run after clicking stop and potentially producing irreversible side effects.
- Fix via-messaging where concurrent delegation/messages/help requests to the same target resulted in replies being incorrectly matched, silently returning wrong results.
- Fix missing correlation IDs in asynchronous send_message receipts, causing cross-path mismatches that incorrectly resolved synchronous waiters.
- Fix marketplace list where all ADR-038 pointer skills were invisible, unsearchable, and unbrowsable.
- Fix marketplace skill version determination: when the upstream had no version, both sides defaulted to 0.0.0, causing updates to be silently skipped forever.
- Fix marketplace frontmatter incorrectly reporting "must be array" for comma-separated allowed-tools/tags strings.
- Fix pointer entries not passing through requiredClientVersion, causing the upgrade gate to always evaluate to false.
- Fix compat-proxy routing requests for models with the same name (e.g., official cloud and local configurations sharing a model name) incorrectly to the local provider.
- Fix scheduled tasks where maxTurns=1 caused the agent to reply only once and then be cut off, never completing the task.
- Fix missing feedback loop for failed scheduled tasks, rendering backoff/alerting/disable mechanisms completely ineffective, causing tasks to repeatedly retry at full intervals.
- Fix relative time drift for the model during conversations, where "in X minutes" and scheduling times gradually deviated from real time in long conversations.
- Fix the relationship graph where "delivery" edges were never rendered (events were silently discarded).
- Fix the five thinking depth levels actually producing only three budget values, making "deep" and "deepest" indistinguishable.
- Fix thinking mode where the Anthropic compatible gateway required thinking blocks in responses; missing blocks caused a 400 error that could not be retried.
- Fix cases where max_tokens for models like deepseek-v4 exceeded the gateway limit, and the resulting Chinese error message could not be recognized by the self-healing mechanism.
- Fix GLM and similar models where tool parameters were emitted as arrays during very large file calls, causing session crashes when going through the gateway.
- Fix hardcoded enumeration of credencialSource causing new credential types to poison older clients and making compute services unavailable.
- Fix configuration center merging of provider presets where the write path lacked salvage protection, causing a single bad provider to poison the entire configuration.
- Fix defaultServiceMap silently discarding entries missing providerId, causing the default model to drift permanently.
- Fix compute configuration seed/damage recovery path with no fallback, causing bad data to prevent agent-service from starting.
- Fix the agent list status indicator being reset to green during tool execution, falsely showing idle instead of the actual busy state.
- Fix AgentFS file watching completely failing because the root directory was hidden, so changes to agent.json never triggered hot updates.
- Fix fan-out delegation skipping target validation, causing non-existent agents to silently run with empty configurations and appear successful.
- Fix delegation cards showing "unknown" for fan-out targets with multiple destinations.
- Fix imported/cloned agents not being registered in the Registry after being saved to disk, causing the list to show offline until restart.
- Fix the chat sidebar list not refreshing after renaming an agent, and the model incorrectly claiming a name change without calling the tool, which was then recorded as a memory.
- Fix residual teams not disbanded after deleting an agent, and duplicate teams with the same name accumulating due to repeated AI creation.
- Fix issues in team shared working directory selection, delegation, context injection, and other paths.
- Fix context trimming token underestimation (missing counts for thinking, tool names, array contents), causing insufficient initial headroom for non-OpenAI models.
- Fix context trimming budget not deducting the fixed system + tool base, and the absence of the ability to unload tool results without loss for re-fetching.
- Fix the message action bar overflowing and covering the timestamp on short messages.
- Fix a horizontal scrollbar appearing at the bottom of the chat message area, causing long URLs/tokens/wide tables to be clipped.
- Fix the whole window flickering when scrolling past the bottom of the message list.
- Fix clicking file:// links causing errors on macOS and gray screen freeze on Windows; now they are opened with the system default app.
- Fix the cursor not scrolling into view after programmatic text changes in the chat input (Shift+Enter line break, history recall, draft recovery, etc.).
- Fix autocomplete selection paths also lacking cursor scroll-follow.
- Fix the message area scrollbar being too narrow to drag.
- Fix multi-select mode where clicking the selection circle was unresponsive, and audio/video areas were inadvertently selected.
- Fix "Clear Data" failing with EPERM on Windows due to read-only git files and reserved device name files.
- Fix Windows git paths containing spaces causing simple-git to crash, breaking all marketplace/Agent git operations.
- Fix the built-in git tarball extraction with MSYS tar on Windows always failing.
- Fix backslashes in Windows paths being swallowed by Markdown escaping, showing incorrect paths.
- Fix the service directory approval window crashing due to dirty data (missing or invalid protocol/status).
- Fix the service directory status not updating in real time, requiring a manual refresh after approval.
- Fix the task board probe returning 404 for every session, causing noise in the browser console.
- Fix conversation-memory messageRange incorrectly passing timestamps, causing the message count to display in the hundreds of millions.
- Fix rrule test time bombs and script executor EPIPE uncaught exceptions causing random CI failures.
- Fix utterance paragraph memories being solidified into facts without tool execution, retroactively contaminating subsequent conversations.
- Fix relationship heatmap floating-point numbers being too long, breaking the card layout.
- Fix the docx seed self-check always failing due to nested shell quoting, causing a first-start interruption.

## Improvements
- Relaxed the delegation sub-task synchronous wait timeout from 5 minutes to 30 minutes. On timeout, the sub-session is no longer aborted; instead, the caller is prompted to make a proactive decision.
- The WorkflowRun tool now has no default timeout; it is an optional parameter controlled by the agent.
- Scheduled task turns increased from 1 round to the same as normal chats; the agent now determines its own boundary.
- Automatic exponential backoff on scheduled task failure (30s → 1 hour), with alerts triggered on consecutive failures.
- Thinking depth level persisted to agent.json, so it survives restarts.
- OpenAI/Anthropic dual-format thinking parameters automatically adapt to each provider's dialect, ensuring compatibility with domestic gateways.
- For compat gateways that do not declare a dialect, thinking activation parameters are fully injected based on the recognized dialect.
- Codex response model explicitly sends reasoning-off parameters when the off level is selected.
- Provider-level requiredClientVersion runtime gate; unsupported providers prompt to upgrade the client.
- Context trimming now uses a cheapest-first cascade (truncate → remove turns → collapse → exceed soft floor), eliminating the deadlock where trimming still exceeds the limit.
- Streamlined and deduplicated the system prompt base, removing invalid spawn_agent capability declarations.
- Enhanced agent configuration compute hardening: API keys are write-only (masked in HTTP responses + file tool protection + Bash blacklist).
- Added a config-center reload endpoint, allowing the frontend to manually refresh and validate compute configuration.
- Claude subscription provider card now shows the subscription tier and connection status, with support for cancellation and sign-out.
- Built-in git is installed with the package, so agent snapshots and other git features work without requiring a system git installation.
- Runtime directory unified to ~/.desirecore; existing users are migrated automatically.
- Video generation relay upgraded to OpenAI Video API format, integrating with Doubao Seedance 2.0.
- Compute service model list now partitioned by text/image/video generation, with billing mode and price displayed.
- Chat message area scrollbar hit zone widened for easier drag.
- The Ls tool now executes directly without approval by default, and when OS permission is denied, it returns helpful guidance instead of a raw error.
- Marketplace repository source saves no longer wait for a full sync cycle, avoiding getting stuck on "Saving" and duplicate repository listings.
- All blanket overwrites are now atomic writes (temp file + rename), eliminating half-written corruption from crashes.
- agent.json reads/writes are now protected by a per-agent lock; concurrent modifications are queued and applied instead of returning a 409 error.
