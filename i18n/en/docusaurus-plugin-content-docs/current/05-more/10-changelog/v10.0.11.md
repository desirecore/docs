# DesireCore v10.0.11

**Release Date**: 2026-03-17

## New Features
- **Automatic Email Rule Processing**: Email rules now support auto-replying to new messages or directly triggering an agent to handle them, enabling incoming mail in Gmail, Outlook, and IMAP inboxes to be processed automatically.
- **File-Based Image Transfer in Chat**: Images are now uploaded as media files before being referenced when sent, making image loading in chats more stable and allowing agents to continue recognizing images and processing images returned by tools.
- **Real-Time Streaming Token Display**: While an agent is replying, you can now see the Token usage for the current conversation update in real time instead of waiting until the reply is finished.
- **Delete Individual Chat History Entries**: The chat history pop-up now supports deleting messages one by one, and the corresponding content will also be removed from the conversation view.
- **SuperDoc Save Button**: A Save button has been added to the SuperDoc toolbar, allowing you to write the current document changes to the file more directly.
- **Beta-Only Model Provider**: Added a provider dedicated to internal beta testing along with a set of preset models, making more test models directly available in eligible environments.

## Fixes
- Fixed an issue where "Jump to Chat Position" in chat history would sometimes navigate inaccurately. It can now locate the corresponding message more reliably.
- Fixed an issue where clicking `.md` file links in SuperDoc would throw an error after restarting or refreshing the app. Document links in historical conversations can now be opened normally.
- Fixed an issue where deleted agents would occasionally still appear in the list after deletion. The agent list now stays in sync with the actual state more quickly.
- Fixed an issue in API auditing where input Tokens often displayed as 0. Actual input usage is now correctly counted and shown when caching is enabled.
- Fixed a routing import issue when email rules triggered agent processing, preventing errors in some automated processing flows.

## Improvements
- The way agent profile data is read has been unified, making the display and retrieval of related configuration content more consistent and stable.
- Agent version information has been separated from the AgentFS spec version, making version display and future marketplace update detection clearer and more accurate.
- The input box now correctly restores platform detection logic, so shortcut key hints on macOS and other systems better match the current device.
