# DesireCore v10.0.15

**Release Date**: 2026-03-22

## New Features
- **Agent Folder Import**: You can now directly select a local folder to import an agent in the desktop version, without needing to package it as a ZIP first. The import process is more convenient and fully compatible with Windows.
- **Built-in MCP Tools by Default**: All agents can now use the built-in browser developer tools capability by default, with no manual configuration required to access the relevant MCP services.
- **SuperDoc Resizable Split View**: The SuperDoc editor now supports dragging to adjust the width of the editing area and chat area, and will automatically hide the preview panel when space is limited, making split-screen editing more flexible.

## Fixes
- Fixed an issue where clicking “Retry” on an old message after switching topics would have no effect, and also resolved a case where media attachments could be lost when retrying.
- Fixed an issue where old messages were not properly cleared when retrying or resending after editing, preventing duplicate content from reappearing in history after a refresh.
- Fixed an issue where the chat history popup could only display the most recent messages; earlier conversation content can now be viewed in full.
- Fixed an issue where usage statistics in streaming responses would restart from 0 after the app was restarted or refreshed.
- Fixed an issue where the active status at the top would continue to display after generation was paused; the interface state now resets immediately after stopping.
- Fixed an issue where `.md` file links in conversations could not be clicked to open after a refresh.
- Fixed an issue where agent avatar text appeared in black on the marketplace page and marketplace detail page, making avatar display clearer and more consistent.
- Fixed compatibility issues in Windows during agent creation, import, and deletion.

## Improvements
- Improved the availability of MCP services. The packaged desktop version now more fully recognizes command-line environments, and tool parameter compatibility has also been improved, reducing call failures.
- Improved stability for large requests and file uploads, making errors caused by oversized requests less likely when handling large context content or larger attachments.
