# DesireCore v10.0.49

**Release Date**: 2026-05-08

![v10.0.49 更新概览](/img/changelog/v10.0.49-en.png)

## New Features
- **Platform-specific update checks**: The client now fetches available versions based on the operating system you're currently using, allowing new versions to appear in update prompts sooner once the build for your platform is ready.
- **Expanded Tongyi Qianwen compute and encoding options**: Added and improved DashScope-related model providers, compute presets, and encoding schemes, making it easier to choose from more available compute configurations when creating or using agents.

## Fixes
- Agent auto-update will no longer incorrectly report “already up to date” when there are network issues, repository permission problems, or failures retrieving remote information. Instead, it will clearly prompt you to check your network or remote repository status.
- Clicking “Check for Updates” manually now fetches the latest remote version information directly, fixing an issue where newly released versions could be temporarily undetectable due to caching.
- When agent auto-update encounters Git conflicts, it now clearly shows the conflicting files and restores the persistent “Conflict” indicator in the chat header, making it easier to locate and resolve issues.
- When auto-update fails, it now provides more accurate messages based on the specific cause—such as no remote repository configured, invalid configuration, or the current directory not being a Git repository—instead of showing the same vague failure message for all cases.
- Git repository initialization is now more stable after creating a new agent, fixing occasional failures when performing Git operations immediately after creation.

## Improvements
- The conflict status next to the agent name is now preserved across restarts and refreshed automatically, so if the conflict still exists, you'll continue to see the reminder after reopening the client.
- Auto-update now prioritizes the target branch configured for the agent when pulling code, reducing update issues caused by branch mismatches.
- Default skills and template configurations related to workflows have been updated, making the out-of-the-box experience better aligned with the product’s current capabilities.
