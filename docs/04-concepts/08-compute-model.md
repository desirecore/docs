# 算力模型

## 什么是算力模型

DesireCore 本身不提供 AI 模型，而是帮你**连接和管理**各种 AI 服务。就像一台电脑本身不产生电力，但它能使用不同的电源 -- 你可以接市电、用电池，甚至太阳能。

DesireCore 的算力模型让你可以自由选择 AI 服务供应商，用自己的 API Key 直接对接，不需要通过任何中间商。

## BYOK -- 自带钥匙模式

BYOK 是 **Bring Your Own Key**（自带钥匙）的缩写，这是 DesireCore 的核心理念之一。

### 怎么理解 BYOK

想象你去一家自助餐厅，但它不卖食材 -- 你自己带食材来，餐厅提供厨房、厨具和烹饪指导。这样做的好处是：

- **你完全掌控食材来源和成本**
- **餐厅不会在食材上加价**
- **你可以随时换供应商**

在 DesireCore 中：

- 你注册各 AI 供应商的账号，获取 API Key
- 把 API Key 填入 DesireCore 的设置中
- DesireCore 直接使用你的 Key 调用 AI 服务
- 费用直接由供应商向你结算

### BYOK 的优势

| 优势 | 说明 |
|------|------|
| **透明计费** | 你直接和供应商结算，没有中间加价 |
| **自由选择** | 今天用 OpenAI，明天换 DeepSeek，随时切换 |
| **数据安全** | 你的数据直接发给供应商，不经过第三方 |
| **成本可控** | 在供应商后台就能看到消费明细 |

## 模型选择策略

不同的任务适合不同的模型。DesireCore 通过**默认映射**帮你自动选择：

| 服务类型 | 适用场景 | 推荐模型特征 |
|---------|---------|-------------|
| chat | 日常对话、复杂任务 | 综合能力强的旗舰模型 |
| fast | 简单任务、快速响应 | 响应速度快的轻量模型 |
| reasoning | 数学、逻辑推理 | 深度推理专用模型 |
| vision | 图像理解 | 多模态模型 |
| embedding | 文本向量化 | 嵌入模型 |
| tts | 语音合成 | TTS 模型 |
| asr | 语音识别 | ASR 模型 |

你可以在设置中自定义这些映射，比如把日常对话指向便宜的模型，把重要任务指向最强的模型。

## 预置算力包

如果你不想自己注册供应商账号，DesireCore 也计划提供预置算力包 -- 开箱即用，按用量付费。但 BYOK 模式始终可用，你不会被锁定在任何单一供应商上。
