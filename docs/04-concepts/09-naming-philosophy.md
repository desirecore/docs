# 命名理念

**DesireCore** = **Desire**（意愿/愿望）+ **Core**（核心）

---

## Desire：意愿与愿望

"Desire"指的是**用户的意图与愿望**，这是产品的核心关注点：

- 产品的核心理念是将用户的**意图、标准、边界、偏好**沉淀为可复用的行为规范
- 当前 AI 的痛点是"意图断裂"——用户的经验和偏好无法沉淀，AI 永远只是"没有记忆的临时工"
- DesireCore 要做的是"把对话中的意图、规则、示例持续沉淀为 Agent 的行为规范"

**Desire 代表用户想要的东西——不只是即时的请求，而是长期的意愿与期望。**

---

## Core：核心引擎

"Core"有两层含义：

1. **操作系统层面**：DesireCore 定位为"Agent OS（委派式交互的 Agent 操作系统）"，是 Agent 的核心运行时系统
2. **沉淀层面**：它是用户意图的"核心沉淀层"——把零散的对话凝结成智能体的核心资产

### 五类核心资产

| 资产 | 说明 |
|------|------|
| Persona（人格档案） | 智能体是谁——风格、语气、价值取向 |
| Playbook（行为手册） | 智能体怎么做事——SOP、规则、边界 |
| Memory（记忆账本） | 智能体记住什么——事实、偏好、上下文 |
| Tool Body（工具身体） | 智能体能做什么——可调用的工具与权限 |
| Receipts（行动回执） | 智能体做过什么——证据链与可回滚点 |

---

## 命名深意

**DesireCore = 意愿的核心 / 愿望之核**

| 视角 | 含义 |
|------|------|
| 用户视角 | 把你的"愿望"变成智能体的"内核"——教会智能体你想要什么 |
| 系统视角 | Agent 操作系统的核心引擎，驱动意图→规划→执行→回执的闭环 |
| 关系视角 | 智能体的"内心"（Core）承载着对用户意愿（Desire）的理解与承诺 |

---

## 与产品定位的呼应

### 一句话概述

> DesireCore 是一套让人类能够**培养并托管 AI 智能体**的交互与治理系统。

### 命名与定位的关联

- **培养**：你不是在给工具下命令，而是在把你的意愿（Desire）注入智能体的核心（Core）
- **托管**：智能体的 Core 承载了你的 Desire，能够在边界内自主规划与执行
- **治理**：Core 是可版本化、可审计、可回滚的，确保意愿的沉淀是可控的

### 交互范式升级

| 传统模式 | DesireCore 模式 |
|----------|----------------|
| 人问 AI 答 | 人教 AI 做事 |
| 下指令给工具 | 培养 AI 智能体 |
| 一次性消费 Prompt | 意愿沉淀为长期规范 |
| Interaction（交互） | Delegation（委派） |

---

## 总结

**DesireCore** 这个名字精准表达了产品的核心价值主张：

> 将用户的意愿（Desire）凝结为智能体的核心（Core），让 AI 从"临时工"进化为"可托管的智能体"。
