# 数据存储位置

DesireCore 将所有数据存储在本地文件系统中，不依赖云端服务器存储你的个人数据。

登录账号后，订阅、credit、云端模型列表等账号资源会从 DesireCore 在线服务同步；本地对话、智能体、文件和自定义 API Key 仍保存在本机。

## 各平台数据根目录

| 平台 | 数据根目录 |
|------|-----------|
| **macOS** | `~/.desirecore/` |
| **Windows** | `用户文件夹\.desirecore\` |
| **Linux** | `~/.desirecore/` |

> `~` 表示用户主目录。例如 macOS 上 `/Users/你的用户名/.desirecore/`

如果设置了 `DESIRECORE_HOME`，DesireCore 会优先使用该路径。开发版和生产版默认隔离：生产版通常使用 `~/.desirecore/`，开发版通常使用 `~/.desirecore-dev/`。

## 目录结构详解

| 目录 | 用途 | 包含内容 | 可否删除 |
|------|------|---------|---------|
| `config/` | 全局配置 | 用户设置、偏好、API Key 配置 | 删除后恢复默认设置 |
| `config/avatar/` | 用户头像 | JPEG/PNG/WebP 格式的头像文件 | 可安全删除 |
| `agents/` | 智能体本体 | 每个智能体一个子目录（Git 仓库） | 删除后失去智能体数据 |
| `agents/desirecore/` | 核心智能体 | DesireCore 内置智能体 | 删除后自动重装 |
| `users/` | 用户数据 | 个人画像、偏好、与各智能体的关系记忆 | 删除后失去个性化数据 |
| `skills/` | 全局技能 | 所有已安装的全局技能包 | 删除后内置技能自动重装 |
| `runs/` | 运行记录 | 会话记录、任务回执 | 可安全删除（丢失历史记录） |
| `cache/` | 缓存 | 索引缓存、关系图谱缓存 | 可安全删除（自动重建） |
| `logs/` | 日志 | 运行日志和错误排查信息 | 可安全删除 |
| `market/` | 市场数据 | 官方市场仓库缓存 | 可安全删除（重新拉取） |
| `registry/` | 注册表缓存 | 模型、服务、市场元数据 | 可安全删除（重新同步） |
| `apps/` / `services/` | 应用与服务 | 已安装应用、派生服务和状态 | 删除后需重新安装 |

## 应用程序目录

DesireCore 应用本身的安装路径：

| 平台 | 应用安装路径 |
|------|-------------|
| **macOS** | `/Applications/DesireCore.app` |
| **Windows** | `C:\Users\你的用户名\AppData\Local\Programs\desirecore\` |
| **Linux** | 取决于 `AppImage` 文件所在的位置 |

## 配置文件

| 文件 | 路径 | 说明 |
|------|------|------|
| 用户配置 | `config/user-profile.json` | 用户基本信息和偏好 |
| 算力配置 | `config/compute/` | API Key 和供应商配置 |
| 快捷键配置 | `config/shortcuts.json` | 自定义快捷键 |
| 通知数据 | `config/notifications.json` | 通知中心状态和未读记录 |
| 密钥引用 | `config/secrets.json` | 凭据引用和授权关系，不应导出明文 |
| 调度状态 | `config/schedules/` 或运行记录目录 | 定时任务配置、触发历史和状态 |

## 备份建议

以下目录包含不可再生的重要数据，建议定期备份：

| 优先级 | 目录 | 原因 |
|--------|------|------|
| **高** | `agents/` | 智能体的所有学习成果和配置 |
| **高** | `users/` | 个人画像和关系记忆 |
| **中** | `config/` | 个人设置和 API Key |
| **中** | `skills/` | 用户自定义的技能 |
| **低** | `runs/` | 历史任务记录 |

你可以通过「设置 > 数据与隐私 > 导出数据」选择性导出数据作为备份。导出时可以按智能体、团队、技能、对话记录、系统配置、媒体、邮件、工作区文件和审计日志等类别选择内容。

## 清除数据

在「设置 > 数据与隐私 > 清除本地数据」中，你可以选择性地清除：

| 数据类别 | 清除范围 | 是否可恢复 |
|---------|---------|-----------|
| 对话记录 | 聊天消息、会话历史、回执 | 不可恢复 |
| 智能体 | 配置、人设、记忆（核心智能体自动重装） | 不可恢复 |
| 技能 | 全局技能（内置技能自动重装） | 不可恢复 |
| 算力 | 供应商配置、模型列表 | 不可恢复 |
| 存储 | S3 或对象存储连接配置 | 不可恢复 |
| 代码托管 | GitHub、Gitee、Gitea 等连接配置 | 不可恢复 |
| 用户档案 | 身份、头像、记忆库 | 不可恢复 |
| 市场数据 | 官方市场仓库缓存 | 自动重建 |
| 缓存与日志 | 索引缓存、日志文件、错误排查信息 | 自动重建 |

审计日志可以在导出数据中作为独立类别导出；当前清除入口没有独立的审计日志类别，相关执行记录会随对话记录、缓存与日志等实际存储位置一并处理。

:::warning 密钥与导出
数据导出不会把系统凭据管理器中的明文 API Key 直接写进备份文件。迁移到新设备后，你可能需要重新授权或重新输入部分密钥。
:::
