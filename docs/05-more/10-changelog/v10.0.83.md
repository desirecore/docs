# DesireCore v10.0.83

**发布日期**: 2026-07-12

![v10.0.83 更新概览](/img/changelog/v10.0.83.png)

## 新功能
- **Claude 订阅接入**：支持通过 Claude Code 原生登录、应用内 OAuth 或手动 setup-token 三种方式接入 Claude Pro/Max 订阅算力，经内置 Claude Agent SDK 后端履约，审计链路完整
- **三层提示词体系**：新增全局层（用户级）、团队层（git 管理随团队分发）和 Agent 层三层叠加提示词，提供集中式「提示词中心」UI 统一管理，支持结构化表单编辑与实时预览
- **思考深度控制**：输入框新增思考深度档位选择按钮（会话级），支持 auto/off/低/中/高/最深 六档，OpenAI 与 Anthropic 双格式自动兼容，档位可持久化到智能体配置
- **Windows GUI 自动化**：Windows 安装包内置 cua-driver 0.7.1，智能体可直接操控 Windows 原生应用（截图、点击、输入、读取窗口状态），支持后台窗口不抢焦点，开箱即用
- **离线运行时开箱即用**：安装包内置 Python 3.13 和 Node.js 24 LTS 运行时，首启自动导入，完全离线可用；版本列表支持离线兜底，消除空列表无限重试问题
- **内置 Git 确定性依赖**：随应用打包 Git 2.53.0，无需系统安装 Git 即可使用市场同步、Agent 快照等功能
- **AgentFS Git 全树快照**：回撤机制从文件级拷贝升级为 Git 全树快照，支持整体还原（含删除后重建）、天然覆盖离回合后台写入
- **独立部署模式**：Agent Service 可脱离 Electron 以独立进程部署到服务器，支持远程安全访问与一客户一容器交付（含实例锁、Token 鉴权、SSRF 防护、Caddy TLS 等完整配方）
- **上下文智能管理**：上下文裁剪改为级联策略（先截工具输出→再删老轮→中段折叠），可重取工具的输出无损卸载；从超限报错自动学习模型真实上下文窗口与输出上限并持久化，大输出一次成型
- **算力计费与供应商公告**：算力模型列表顶部展示「算力由合作伙伴提供」公告，模型按文本/图片生成/视频生成分区展示（生成类标注「对话不可用」），价格按 token/按次等计费模式分支展示
- **文件「打开方式」下拉**：消息中的文件条目新增「打开方式 ▾」下拉菜单，支持应用内打开、系统默认应用打开、选择其他应用打开、在文件夹中定位显示
- **视频生成升级**：视频生成中转升级为 OpenAI Video API 兼容路径，接入官方云算力开箱即用 HappyHorse 视频生成，同时适配豆包 Seedance 2.0 视频模型
- **迁移到新设备**：新增「迁移到新设备」完整导出导入向导，支持 M1 身份替换（身份跟人走）和 M2 归并到当前身份（源数据定向重映射到本机身份）
- **迁移 API 密钥**：迁移导出可选加密打包 API 密钥（scrypt + AES-256-GCM），导入时支持逐 Provider 替换/合并决策，避免新设备重新配置所有密钥
- **套餐折扣展示**：套餐卡片价格区支持原价划线与省 X% / 限免折扣徽章展示
- **手动刷新服务目录**：「应用与服务」页面新增手动刷新按钮，点击即时同步注册表目录
- **算力预置合并冲突保护**：config-center 预置更新时，用户已修改的 Provider 字段（地址/标签/格式等）永不被上游覆盖，冲突项通过审阅面板可视化对比决策
- **Provider 级版本门槛**：支持 config-center 为 Provider 声明客户端最低版本要求，不满足时提示「需更新客户端」而非静默不可用
- **Provider 并发限流**：支持为每个 Provider 配置最大并发请求数与排队上限，超出排队等待而非报错，前端实时显示排队状态
- **排队查询可中止**：排队等待中的查询支持点击停止立即出队，不再被无条件调度执行产生副作用
- **算力预置配置版本号**：算力页面头部显示当前预置数据版本号，便于快速确认数据同步状态
- **Claude SDK 会话恢复**：Claude 订阅路径支持 session-resume 恢复 prompt cache，减少重复上下文的 quota 消耗；支持图片在订阅路径直传给模型
- **子任务超时不再自动中断**：委派/生成子 Agent/发送消息的同步等待默认超时放宽至 30 分钟，超时不再自动中止子会话，改为告知调用方自行决策继续或主动终止
- **关系图谱交付边**：关系图谱补齐「交付」关系边，Agent 成功交付结果后图谱可正确渲染交付连线
- **对话 PDF 导出客户端渲染**：对话导出 PDF 改为客户端端渲染，消除对 Agent Service 的宿主依赖
- **扫描件遇非视觉模型智能提示**：非视觉模型读取纯扫描件 PDF 时，不再清空工具导致无限复读，改为服务端直接提示「请切换到支持视觉理解的模型」并结束本轮
- **SSRF 防护下沉到 per-Agent**：SSRF 防护从全局开关扩展为逐 Agent 可配置，修改即时生效无需重启，全局与 Agent 级取更严
- **Ls 工具执行优化**：Ls 列目录默认免审批直接执行（只读操作），macOS 首次访问受系统权限管控的目录时自动弹原生授权框，权限被拒时返回明确的系统设置引导
- **市场列表纳入 pointer 技能**：ADR-038 pointer 技能在市场浏览/搜索/详情中完全可用，版本判定修复（无 version 字段时回退市场登记版本），frontmatter 校验兼容逗号分隔写法
- **消息区滚动条加宽**：聊天消息区垂直滚动条轨道加宽至 10px，方便鼠标拖拽
- **导入/克隆智能体立即可用**：通过任意方式导入或克隆的智能体落盘后立即在线，不再需要重启客户端
- **委派提示词路径规则**：委派任务时智能体会主动使用绝对路径引用文件/目录，避免接收方工作目录不同导致路径解析失败

## 修复
- 修复 macOS 构建因内置 Python 运行时二进制签名缺 secure timestamp 被 Apple 公证拒绝的问题（macOS 包首启回退在线安装运行时）
- 修复 Windows 构建因 asar 路径分隔符导致 SDK 文件校验误报全部 miss 的问题
- 修复 cpSync 默认行为导致内置 Git 符号链接被重写为绝对路径，造成 macOS 公证失败和 Linux 包内悬空链接的问题
- 修复大文件发布上传时因慢链路传输超过 SigV4 时间窗导致 403 RequestTimeTooSkewed 死循环的问题（≥64MB 文件改用 S3 分片上传）
- 修复 CI 中 detect 与 post-release job 被调度到 macOS runner 导致 gh CLI 下载失败、整体构建 skipped 的问题
- 修复 v10.0.83 构建中多个 after-pack 平台 bug 导致的连续构建失败
- 修复同名模型经代理路由时可能串台到错误 Provider 的问题（多 Provider + 同名模型的精确路由，同时覆盖内部 LLM 调用全路径）
- 修复 compat 未声明方言的网关上选择思考深度不发送开启参数的问题（按方言全量注入）
- 修复非视觉模型读取纯扫描件 PDF 时清空所有工具导致退化复读的死循环
- 修复定时任务只回一句话就不再执行的问题（maxTurns 放开至 30、interim-ack 检测续跑、失败回退闭环）
- 修复定时任务执行结果未反馈导致退避/告警/禁用机制全部失效的问题
- 修复会话中相对时间漂移（模型的「现在」冻结在会话起始时刻，导致「X 分钟后」不准确）的问题
- 修复点击 Agent 回复中 file:// 链接导致 macOS 报错或 Windows 灰屏卡死的问题
- 修复 Windows 下清除数据失败的问题（Git 只读文件和保留设备名文件导致 rmSync EPERM）
- 修复导入的智能体在列表中显示为离线的问题（导入路径落盘后未注册 Registry）
- 修复 AgentFS 文件监听热更新完全失效的问题（chokidar 误将 ~/.desirecore 隐藏目录整树忽略）
- 修复 fan-out 并行委派跳过目标校验导致不存在的目标被静默执行的问题
- 修复委派卡片在 fan-out 模式下显示「unknown」的问题
- 修复委派/消息/协助请求并发时回复错配的问题（via-messaging 路径新增 correlationId 精确匹配）
- 修复 send_message 异步回执因缺 correlationId 可能跨路径错配同步等待方的问题
- 修复聊天输入框按 Shift+Enter 换行时新行落在可视区外、用户误以为换行无效的问题
- 修复聊天消息区滑到底继续滑动导致整窗闪烁的问题（补充 overscroll-contain 阻断滚动链冒泡）

## 改进
- Windows 安装包内置 cua-driver，用户无需手动安装即可使用 GUI 自动化能力
- Windows Git 路径自动转换为 8.3 短路径以消除 simple-git 警告噪音
- 聊天消息区滚动条命中区加宽至 10px 方便拖拽
- 任务板详情探测改为非 404 语义消除启动和切换会话时的控制台噪音
- 市场仓库源保存解耦整轮同步，修复添加仓库后长时间卡「保存中」和仓库显示两份的问题
- 应用与服务页面新增手动刷新按钮，方便即时同步注册表目录
- 算力页面头部显示当前预置配置版本号
- 套餐卡片支持原价划线与折扣展示
- 套餐价格按 token/按次等计费模式分支展示
- Cloud 模型按文本/图片生成/视频生成分区展示，生成类模型标注「对话不可用」
- 算力供应商公告由接口数据驱动，去掉写死的合作伙伴名
- 运行时目录统一到 ~/.desirecore，老用户自动迁移
- DNS rebinding 防护补齐（外呼前 DNS pinning 复核 + undici 连接锁定）
- 邮件中心与工作流等集成功能的权限和安全加固
- CI 门禁合并优化（Typecheck 与 Unit Tests 合并为单一 Job + tsbuildinfo 增量缓存）
- CI 关闭 setup-node 默认 npm 缓存，消除自托管 runner 巨量出网流量
- standalone smoke 触发范围收紧，仅在 lib/\*\* 等核心路径变更时执行
- 依赖升级：Electron 43.1.0、Node.js 内置升级、fastify 5.10.0、tiptap 全家桶 3.27.3、typebox 1.3.5 等
