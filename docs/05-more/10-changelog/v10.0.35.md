# DesireCore v10.0.35

**发布日期**: 2026-04-21

![v10.0.35 更新概览](/img/changelog/v10.0.35.png)

## 新功能
- **Windows PowerShell 内置工具**：Windows 用户现在可以让智能体直接使用 PowerShell 执行命令与脚本，并支持审批、超时控制、后台运行及大输出处理。
- **输入框继续生成**：当回复被中断且消息气泡中不便操作时，你可以直接在输入框旁一键继续生成，补全对话更顺手。

## 修复
- 修复了 Windows 上 PowerShell 工具无输出、中文乱码、只读命令误触发审批以及不同版本语法提示不准确的问题。
- 修复了内置命令输出处理中会误删 emoji 和多语言字符的问题，现在中日韩文、阿拉伯文、希伯来文及表情符号都能正确保留。
- 修复了暂停生成或清空对话后，聊天头部仍错误显示“工作中”的状态残留问题。
- 修复了切换到已有未读消息的会话后，侧边栏未读红点不会及时消失的问题。
- 修复了 Windows 部分环境下邮件中心因服务端口动态变化而无法连接、数据加载失败或 WebSocket 重复连接的问题。
- 修复了 Windows 部分环境下工作流服务因端口冲突或启动时序问题导致无法访问、请求落到错误端口的问题。
- 修复了部分模型在开启思考模式后仍无法显示思考过程的问题，智谱 GLM 及相关接入点现在可以正常展示思考内容。
- 修复了长上下文场景下对话预算估算不准的问题，减少了上下文超限却未及时自动压缩而影响回复的情况。
- 修复了模型预置同步后旧模型残留、排序异常以及已弃用模型仍出现在聊天模型列表中的问题。

## 改进
- 对话流式生成期间的界面刷新更加流畅，减少了后台会话更新对当前聊天界面的干扰。
- 侧边栏消息预览改为在消息完成或新增后更新，换来更稳定的整体聊天性能。
- 内测模型列表已更新，默认模型调整为更优先的高速版本，并恢复提供 MiniMax-M2.5 作为备选模型。
