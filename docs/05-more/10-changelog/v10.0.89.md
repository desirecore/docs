# DesireCore v10.0.89

**发布日期**: 2026-07-17

![v10.0.89 更新概览](/img/changelog/v10.0.89.png)

## 新功能
- **文件安全逐智能体开关**：文件访问保护现在默认为不拦截，支持在智能体配置中按需开启工作目录限制，避免对日常操作造成过度影响。
- **检查点治理面板**：在「设置 → 数据与隐私」中新增「回撤与检查点」管理页面，可查看检查点用量、配置保留期限、一键安全清理已归档的旧数据，并控制各类型操作的历史记录开关。

## 修复
- 修复了委派任务或创建子会话时，文件编辑等操作的审批卡片缺少进度条且无法自动决策，导致流程永久卡死的问题。
- 修复了 macOS 上因系统 Keychain 权限弹窗导致启动卡死，并引发本地服务连接永久报错、智能体列表无法加载的致命问题。
- 修复了侧边栏智能体列表加载失败后，点击重试按钮偶尔无响应的问题，同时在自动重试期间增加了“自动重试中”的友好提示。
- 修复了在通过某些 API 网关（如 new-api 转发 DeepSeek）时，所有带工具调用的请求都会失败并返回 400 错误的问题。
- 修复了在同一轮对话中，智能体先添加工作目录再读写文件时，可能因工具执行顺序错乱而失败的问题。
- 修复了在多轮对话中，文件读写权限未能及时更新或错误残留旧的精确授权，导致操作被意外拒绝或放行的问题。
- 修复了智能体画像页面中，部分普通标题和说明文字被错误渲染为标签的表现层瑕疵。
- 修复了在对话回撤检查点预览中，部分使用相对路径写入的文件被误报为“未备份”的虚警问题。
- 修复了在任务委派或跟进对话等场景下，聊天界面的多个消息气泡会彼此堆叠重叠，导致内容难以辨认的问题。
- 修复了在遭遇网络异常、上下文溢出等故障触发对话重试时，已成功执行的工具调用（如写入文件）可能被重复执行而产生副作用的关键问题。
