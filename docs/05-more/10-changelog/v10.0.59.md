# DesireCore v10.0.59

**发布日期**: 2026-05-15

![v10.0.59 更新概览](/img/changelog/v10.0.59.png)

## 新功能
- **沉浸式语音交互**：新增更完整的语音对话能力，你现在可以在沉浸式模式下使用持续语音输入、实时语音识别、语音播报与打断，获得更自然的口语化交互体验。
- **AI 下一条消息预测**：助手回复结束后，输入框上方会智能给出你可能想发送的下一句话建议，可通过点击或快捷键快速采纳。
- **/steer 运行中引导命令**：当智能体正在执行任务时，你可以用 `/steer` 追加新的引导指令，让它在不中断当前流程的情况下调整后续行为。
- **心跳通知优化**：新增顶部心跳通知栏，并支持将连续心跳消息折叠展示，让你更容易查看当天动态而不被刷屏。
- **项目技能目录升级**：项目现在优先识别 `.agents/skills` 中的技能，同时继续兼容旧的 `.claude/skills` 目录，方便新旧项目平滑过渡。

## 修复
- 修复了 Windows 自动更新在部分环境下反复提示“无法卸载旧版本文件”的问题，升级流程更加稳定可靠。
- 修复了智能体同步冲突对话框中“保留本地”可能把冲突标记一并保存的问题，合并结果现在更准确。
- 修复了智能体同步冲突时本地/远端内容对比显示异常的问题，差异预览现在更清晰可读。
- 修复了冲突解决弹窗在未完成处理时尝试关闭会卡住的问题，并补上“保留两者”这一更直观的处理入口。
- 修复了聊天输入框按上键回看历史时，连续相同消息会重复出现的问题。
- 修复了创建或编辑 Markdown 文件后，某些情况下不会显示附件卡片的问题，现在生成的文档更容易直接打开查看。
- 修复了聊天中偶发出现只有头像和时间戳的空白助手消息气泡的问题。
- 修复了运行状态面板中的技能名称和描述不会跟随界面语言切换的问题。
- 修复了快速切换语言或智能体时，技能列表偶尔显示成旧内容的问题。
- 修复了“检查更新”后界面可能一直停留在“检查中”的问题。
- 修复了手动版本切换弹窗在重复打开时残留上次数据、加载状态不明显和提示信息混乱的问题。
- 修复了合并消息视图里反复编辑同一条用户消息后旧版本仍残留的问题。
- 修复了截断聊天历史后，旧的话题记忆仍可能继续影响后续对话的问题。
- 修复了算力配置页面在某些情况下加载失败并返回错误的问题。
- 修复了首次启动解压内置资源后，部分环境下服务无法正常启动的问题。
- 修复了证书过期时本地安全连接可能异常的问题，避免因证书续期时机导致服务不可用。

## 改进
- 技能、市场、配置中心和应用注册表的内置内容现在统一以完整离线资源包随应用提供，首次安装和断网场景下也能更完整地使用默认内容。
- 默认全局技能已恢复离线可用，未联网时也能看到并使用内置技能。
- 审计日志不再自动清理，并恢复记录流式输出细节，方便你保留更完整的排查历史。
- 算力服务配置已更新并对齐多家模型供应商的最新能力，模型可用性与媒体技能适配更完整。
- 新增小米 MiMo 语音合成能力，并补充了更多语音与媒体相关服务支持。
- 模型验证结果现在会被保存，未通过验证的模型会在选择时自动隐藏，减少误选无效模型的情况。
- Markdown 文件输出体验得到优化，相关内容不再被错误自动链接，文档生成与查看路径更清晰。
- 工作流校验更严格了：缺少关键字段、字段名写错或条件分支无法继续时，会更早明确提示，避免运行后才发现问题。
- 内置技能同步机制更高效，减少了不必要的重复扫描，让启动和后台同步过程更轻量。
- 搜索与打包链路得到优化，生产环境下文本搜索工具可正常使用并保持更好的性能。
- 多项底层依赖已升级，带来更好的安全性、稳定性和兼容性。
