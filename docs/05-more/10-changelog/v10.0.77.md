# DesireCore v10.0.77

**发布日期**: 2026-06-15

![v10.0.77 更新概览](/img/changelog/v10.0.77.png)

## 新功能
- **对话记录导出**：现在可以将整段对话或选中的消息导出为 Markdown 或 PDF，并可按需选择是否包含工具调用、思考过程和系统消息。
- **ChatGPT 订阅接入**：现在可直接复用本机 ChatGPT/Codex 订阅凭证使用相关模型，无需额外填写 API Key。
- **付费与余额中心**：现在可在客户端购买算力 credit 包，并在个人资料、侧边栏和资源面板中查看余额及临近过期额度提醒。
- **团队浮层与设置入口**：会话列表中的团队 TL 徽章现支持展开成员浮层，可快速查看成员、进入对应聊天并打开团队设置。
- **最低版本升级引导**：当当前客户端版本低于官方最低支持版本时，应用会给出明确的升级提示并引导你检查更新。

## 修复
- 修复了带附件的用户消息在“重试”后，附件会残留在输入框上方的问题。
- 修复了编辑用户消息时气泡宽度突然变窄、文字显示不完整的问题。
- 修复了导入智能体或团队后，侧边栏不会立即出现新内容、需要手动刷新才能看到的问题。
- 修复了仅勾选“关系”或“根级偏好”时克隆智能体可能失败的问题。
- 修复了主控智能体仍可被克隆的异常情况，避免误操作影响系统核心角色。
- 修复了通过 Compat Proxy 使用 DeepSeek 等模型时，部分请求因角色字段不兼容而报错的问题。
- 修复了委派执行后智能体偶发陷入重复输出、刷屏或静默失败的问题。
- 修复了开发环境重启后前端偶发无法自动连上实时通道、导致审批卡片不弹出等问题。
- 修复了“应用与服务”中部分官方 HTTP 服务显示了不可用的“测试可达性”按钮并报内部错误的问题。
- 修复了“清除本地数据”在开发模式或自定义数据目录下看似成功但实际未清除的问题，并避免清理过程异常时界面一直卡在加载状态。
- 修复了已安装应用无法派生服务的缺口，相关应用现在可以正常生成可用服务项。
- 修复了技能列表在切换界面语言后，名称和描述不会随之本地化更新的问题。

## 改进
- **PDF 阅读体验**：读取 PDF 时会更智能地逐页处理长文档和扫描件，支持继续续读，并在可用时直接借助视觉模型理解扫描页内容。
- **应用与服务目录体验**：现在可以直接点进派生服务详情、手动重新派生服务，并实时看到待审批服务数量变化。
- **Linux 启动安装**：Linux AppImage 版本新增更稳定的启动器安装支持，便于创建命令入口和桌面快捷方式。
