# DesireCore v10.0.51

**发布日期**: 2026-05-11

![v10.0.51 更新概览](/img/changelog/v10.0.51.png)

## 新功能
- **冲突解决对话框**：当智能体同步出现冲突时，你现在可以直接在应用内查看本地与远端差异、选择保留哪一方或手动编辑内容完成解决，不再只能看到文件名提示。
- **对话压缩命令**：新增 `/compact` 手动压缩对话功能，你可以在不清空当前会话的情况下将较早内容压缩为摘要，保留最近几轮上下文继续聊下去。
- **技能多语言注入**：智能体现在会根据你的客户端语言自动注入对应语言的技能说明，在中文界面下可获得更一致的中文技能体验。
- **YAML Frontmatter 渲染**：在编辑 `SKILL.md` 等文件时，YAML frontmatter 现在会被正确识别并以结构化样式展示，不会再被误渲染成分隔线。
- **项目级技能目录升级**：项目中现在支持优先从 `.agents/skills` 发现技能，同时继续兼容旧的 `.claude/skills` 目录。
- **心跳通知增强**：新增顶部心跳通知栏，并支持将连续的心跳消息自动折叠显示，让对话区更清爽。
- **小米 MiMo 能力接入**：新增小米 MiMo Provider 与 TTS 能力支持，可在相关配置中使用新的语音能力。

## 修复
- 修复智能体同步冲突时状态丢失的问题，更新失败会明确提示冲突文件，且冲突标记在重启后仍能正确显示。
- 修复“检测更新”在网络异常时误报“已是最新版本”的问题，主动检查也会绕过缓存，结果更准确。
- 修复冲突解决对话框中的多个关键问题，包括“保留本地”可能提交冲突标记、“关闭后卡住”以及无法方便保留双方内容等情况。
- 修复智能体创建后 Git 仓库初始化的竞态问题，避免新建后立即进行 Git 操作时偶发失败。
- 修复 Agent 写入记忆文件时误弹出 Markdown 编辑器的问题，记忆内容现在会按预期以记忆卡片展示。
- 修复后台心跳重试消息误插入当前用户对话的问题，避免聊天窗口出现无法清除的重试提示。
- 修复工具结果合成格式导致的上游图片数据错误，减少部分模型调用中出现的异常报错。
- 修复 Agent 单次任务可能无限循环调用工具的问题，并减少模型输出重复文本不断累积的情况。
- 修复 `/compact` 在“无需压缩”场景下提示一直转圈不消失的问题。
- 修复大体量代理审计日志下加载缓慢的问题，超大日志场景下列表和统计响应明显更快。
- 修复更新下载可能卡在 0% 不动的问题，并在主源不可用时更可靠地切换到备用源。
- 修复聊天输入框按上键回溯历史时，相邻重复消息会被反复回放的问题。
- 修复社会化关系图谱同步不及时的问题，新安装智能体、用户 @ 提及及关系时间线更新都更加实时。
- 修复媒体技能与算力配置中部分 provider 解析和模型验证问题，避免禁用供应商、Token Plan 或媒体服务场景下匹配异常。
- 修复更多菜单中“检测更新”缺少加载反馈的问题，点击后会立即显示进行中状态并避免重复触发。

## 改进
- 智能体“检测更新”现在会更早看到当前平台可用的新版本，无需等待所有平台构建完成。
- 心跳通知卡片支持长内容折叠展开，且文件保存路径展示与打开体验更清晰。
- 代理审计现在默认以更轻量的方式展示摘要，详细内容按需加载，在保留排障信息的同时减少卡顿。
- 代码托管面板中的“添加仓库”文案已调整为“添加托管源”，术语更清晰，减少与市场仓库概念混淆。
- 开发版与正式版现在可在同一台机器上独立运行，彼此不会互相占用数据目录或运行实例。
- 记忆写入规则进一步收紧，智能体会被明确约束将记忆写入 AgentFS，而不是当前工作目录。
