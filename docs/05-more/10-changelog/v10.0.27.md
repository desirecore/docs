# DesireCore v10.0.27

**发布日期**: 2026-04-10

![v10.0.27 更新概览](/img/changelog/v10.0.27.png)

## 新功能
- **邮件全局内建技能**：所有 Agent 现在默认具备邮件操作能力，可直接处理 Gmail、Outlook 和 IMAP 邮箱的收发、搜索、草稿、标签分类、附件下载及自动化规则。
- **编辑器内联变更审阅**：当 Agent 修改文档时，你可以在编辑器内直接看到新增与删除内容，并按段落逐条接受或拒绝，无需切换到单独的差异视图。
- **选择性数据导入导出**：你现在可以按智能体、技能、对话记录、系统配置、媒体、邮件和工作区文件等类别自由选择要备份或恢复的内容，并在导入前看到数据体积与冲突提醒。
- **运行环境支持容器检测**：运行环境页面新增对 Docker 和 Podman 的识别与展示，方便你查看本机容器相关环境是否可用。

## 修复
- 模型服务返回空响应或异常格式错误时，对话中现在会更稳定地显示可读的报错信息，而不再出现无提示或直接展示原始 JSON 的情况。
- 修复了创建团队时新导入的智能体不显示在成员列表中的问题，现在无论在线或离线都可以正常选择。
- 修复了 Windows 下因中文路径导致数据导入失败的问题，包含中文目录名的备份包现在也能正常导入。
- 修复了聊天中点击文档链接时相对路径解析不正确的问题，现在可以更可靠地从对应 Agent 的工作目录打开文件，并在失败时给出明确提示。
- 修复了分页加载和合并消息时可能出现重复消息的问题，对话记录浏览更加准确。
- 修复了运行环境中已安装 Node.js 版本可能显示不出来的问题，安装后的版本列表现在能正确识别与展示。
- 修复了运行环境版本弹窗在界面边缘被遮挡的问题，弹窗位置显示更完整。

## 改进
- 运行环境中的 Node.js、Python 工具链下载在国内网络下更稳定，新增多节点代理与直连回退后，安装成功率更高。
- 运行环境页面的安装、删除等操作现在会显示更及时的实时进度输出，整体使用过程更顺畅。
- 数据导出会自动排除密钥、设备标识、调度状态等敏感或无关内容，备份更安全也更完整。
- 数据导入导出时会自动跳过工作区中的大型运行时目录，如依赖包和虚拟环境，减少备份体积并降低导入失败风险。
- 当你让 Agent 进行写作、编辑文件或生成文档时，系统会更准确地识别你的意图并优化提示词，从而提升内容生成与文件操作的执行效果。
- PDF 相关能力已升级，文档查看、文本选择、字体兼容和整体渲染体验进一步提升。
