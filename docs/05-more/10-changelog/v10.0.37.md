# DesireCore v10.0.37

**发布日期**: 2026-04-25

![v10.0.37 更新概览](/img/changelog/v10.0.37.png)

## 新功能
- **审批通知系统**：当 Agent 发起命令执行审批时，你现在可以通过系统通知、应用内提示和通知中心同时接收提醒，并可直接批准、拒绝、忽略或跳转到对应对话处理。
- **记忆写入卡片**：当 Agent 把信息写入记忆或用户档案时，对话中会直接显示独立的记忆卡片，清楚展示记忆内容、类型、作用域和变更情况，而不再自动打断你打开编辑器。
- **审计日志导出**：在导出或迁移数据时，你现在可以单独选择是否包含审计日志，便于备份排查记录而不混入其他缓存数据。

## 修复
- 修复了部分情况下 Agent 明明已记住的信息在开启新会话后又“忘记”的问题，现在写入用户域记忆的内容也能在后续对话中正常被重新读取。
- 修复了网络波动、限流或流式响应异常时仍反复显示旧错误、重试后消息重复或上下文丢失的问题，使失败后的恢复更加稳定自然。
- 修复了稳定版用户可能误收到测试版更新通知、点击“立即重启”却无反应的问题，更新通道识别现在更准确。
- 修复了审批通知在通知中心看不到待处理项、按钮点击无效、超时后不自动撤回等问题，整个审批处理链路现已正确同步。
- 修复了部分本地算力配置被异常数据污染后无法自动恢复的问题，升级后受影响的模型参数会自动重新校正，无需手动处理。
- 修复了记忆卡片在合并视图中不显示、部分 diff 内容导致界面报错，以及写入未变更内容时状态展示不准确的问题。
- 修复了 `profile.md` 等用户档案写入场景的识别问题，现在这类更新也会正确以记忆卡片呈现而不会误触发编辑器打开。

## 改进
- **记忆写入体验优化**：记忆卡片现在会更准确保留工具执行顺序，并以更友好的中文标签展示“Agent 记忆 / 私有记忆 / 共享记忆 / 用户档案”等作用域及记忆类型。
- **记忆规则统一**：所有 Agent 现在都会自动获得统一的记忆写入规则提示，减少重复说明，也让“该记到哪里、如何分类”更一致。
- 审批提醒现在会显示发起请求的 Agent 名称，并带有更直观的倒计时进度与跳转交互，处理请求时更容易分辨来源和紧急程度。
- 路径输入对 `~` 和常见环境变量的兼容更好，复制粘贴不同平台的文件路径时更不容易因为格式差异而失败。
