# DesireCore v10.0.52

**发布日期**: 2026-05-12

![v10.0.52 更新概览](/img/changelog/v10.0.52.png)

## 新功能
- **沉浸式语音交互**：现在可以在沉浸式模式下进行更完整的语音对话，支持语音输入识别、语音播报、持续监听与字幕显示，整体交流更接近实时口语互动。
- **AI 下一条消息预测**：助手回复后，输入框上方会出现可一键采纳的“猜你下一句”建议芯片，支持点击或按 `Tab/→` 快速填入。
- **/steer 实时引导命令**：当智能体正在执行任务时，你可以用 `/steer` 在不中断任务的情况下插入新的引导指令，让智能体按你的新方向继续推进。
- **/compact 对话压缩**：新增 `/compact` 手动压缩命令，可在保留当前话题连续性的同时压缩旧上下文，让长对话更容易继续进行。
- **智能体冲突解决器**：当智能体同步遇到代码冲突时，现在可以直接在图形界面中查看差异、选择保留本地或远端内容，或手动编辑后解决冲突。
- **YAML Frontmatter 可视化编辑**：编辑 `SKILL.md` 等文档时，YAML frontmatter 现在会被正确识别并以更直观的结构化样式展示，而不会误显示成分隔线。

## 修复
- 修复了证书过期或服务启动顺序异常时，本地安全连接可能失效的问题，应用启动和本地服务通信更稳定。
- 修复了首次启动或打包后内置资源仓库初始化失败的问题，避免出现默认市场、配置或注册表内容无法加载的情况。
- 修复了算力配置页面在部分情况下打开即报错的问题，现在访问算力配置和节点信息不会再出现 500 错误。
- 修复了 Windows 上自动更新时旧版本文件无法卸载的问题，减少升级过程中安装失败的情况。
- 修复了设置页点击版本号后可能一直停留在“检查中”的问题，现在更新检查结果会正常返回。
- 修复了下载更新时可能卡在 0% 不动的问题，并在主下载源异常时更可靠地切换到备用源继续下载。
- 修复了手动版本切换弹窗加载状态不明显、提示版本混乱和再次打开残留旧数据的问题。
- 修复了智能体同步检测更新时，网络失败却被误报为“已是最新版本”的问题。
- 修复了智能体自动更新发生冲突时只显示泛化失败提示的问题，现在会明确提示冲突并保留冲突状态。
- 修复了冲突解决对话框中“保留本地”可能把冲突标记一并提交的问题，并补上“保留两者”等更实用的处理方式。
- 修复了心跳或后台任务重试提示误插入当前聊天窗口的问题，避免无关重试消息污染对话。
- 修复了 `.md` 文件生成后偶发自动打开方式不正确、附件卡片缺失以及空白助手气泡的问题。
- 修复了内存写入类 Markdown 文件误触发编辑器弹出的情况，记忆内容会按预期以内存卡片呈现。
- 修复了聊天输入框按上键回看历史时，连续重复消息会被反复回放的问题。
- 修复了 `/compact` 在无内容可压缩时提示一直转圈不消失的问题。
- 修复了长时间运行后会话资源可能持续堆积的问题，降低长期使用下的内存占用风险。
- 修复了某些工具结果配对异常时触发上游模型报错的问题，提升复杂工具调用场景的稳定性。
- 修复了单次任务中智能体可能无限循环调用工具、并伴随重复文本越积越多的问题。
- 修复了创建智能体后紧接着进行 Git 操作时可能失败的问题，新的智能体仓库可更稳定地立即使用。
- 修复了心跳通知中使用 `~` 路径时文件被错误写到工作目录的问题，相关文件现在会保存到正确位置。

## 改进
- **项目技能目录兼容升级**：项目级技能现在优先识别 `.agents/skills`，同时继续兼容旧的 `.claude/skills`，让新旧项目都能平滑使用。
- 智能体会根据你的客户端语言优先注入对应语言的技能内容，中文界面下不再频繁收到英文技能指令。
- 系统进一步限制“记忆”写入位置，避免智能体把记忆内容误写入当前工作目录。
- 社会化关系图谱、时间线和权重趋势现在会更快同步，新增智能体、`@` 提及和关系变化能更及时显示。
- 心跳通知体验得到优化，连续消息会自动折叠，并新增顶部通知栏和更清晰的历史展示。
- 心跳历史会自动汇总为可查阅的历史文件，方便后续让智能体回顾过往心跳记录。
- 代码托管面板中的“添加仓库”文案统一调整为“添加托管源”，表达更准确。
- 智能体更多菜单中的“检测更新”现在会立即显示加载状态，减少重复点击带来的困惑。
- 手动版本切换弹窗更宽、更清晰，下载中的目标版本说明也更直观。
- 开发版与正式版现在可以在同一台设备上并行运行，彼此数据与应用身份互不干扰。
- 默认内置市场、注册表、算力配置和核心仓库改为统一的离线引导方式，首次安装在离线场景下也能获得更完整的初始内容。
- 默认全局技能与内置市场内容得到补全与同步，离线状态下的可用性更好。
- 审计日志默认保留完整历史，并恢复记录流式输出细节，方便后续追溯与排查。
- 算力服务配置同步更新，补充了更多供应商与 Token Plan 支持，并优化了模型可用性校验与筛选逻辑。
- 新增小米 MiMo 语音合成能力，并改进了相关媒体与图像技能的服务匹配逻辑。
