# DesireCore v10.0.45

**发布日期**: 2026-05-03

![v10.0.45 更新概览](/img/changelog/v10.0.45.png)

## 新功能
- **心跳检查独立管理**：现在可在智能体设置中单独开启和编辑心跳检查内容，并可一键跳转编辑 `HEARTBEAT.md`，不再需要把心跳当作自动化任务来配置。
- **长会话消息列表虚拟化**：超长对话现在只渲染当前可视区域附近的消息，打开大体量历史会话和来回切换时会明显更流畅。
- **Workflow 服务能力扩展**：工作流服务现已支持更完整的 HTTP 运行与回调链路，让包含代码、Python 代码和大模型节点的工作流执行更完整稳定。

## 修复
- 心跳检查现在会正确使用当前智能体自己的模型配置，避免因模型配置错位导致的鉴权失败或无法执行。
- 修复了心跳检查有时不调用工具、结论提取不准确的问题，通知中现在会更稳定地只显示最终结论而不会混入工具输出。
- 修复了修改心跳间隔或重启应用后上次检查时间丢失的问题，心跳状态恢复更准确。
- 修复了通知中心按钮在桌面端窗口中偶尔无法点击的问题，交互操作更可靠。
- 修复了切换对话时智能体档案面板不会自动收起的问题，界面切换更干净。
- 修复了 RepoChatPanel 的回复路由与文件上下文注入问题，现在围绕 `HEARTBEAT.md` 等当前文件的对话编辑更准确。
- 修复了前端被快捷键或标题栏入口误刷新的问题，避免会话界面被意外重载打断。

## 改进
- 心跳检查机制整体升级，新增失败重试、指数回退、重复内容去重、崩溃恢复和卡住自动恢复等能力，日常提醒会更稳定也更少打扰。
- 心跳结果现在简化为“正常 / 告警”两种状态，状态面板和通知展示更直观，判断成本更低。
- 心跳与自动化调度已彻底分离，自动化面板将只展示真正的任务调度，管理结构更清晰。
- 默认心跳行为更符合直觉：新建或启动后会自动准备心跳配置文件，但仅在你明确开启后才真正执行检查。
- 空白或未配置检查内容的心跳不会触发模型调用，可减少无效检查与不必要的算力消耗。
- 长会话首次进入时改为按需加载最近消息，并分阶段补全内容，首屏出现速度更快。
- 加载更早历史消息时会更稳定地保持当前阅读位置，滚动条拖拽时也不容易被自动滚动打断。
- 会话预览和侧边栏排序逻辑得到优化，长会话在懒加载过程中不容易出现摘要不准或列表跳动。
