# DesireCore v10.0.36

**发布日期**: 2026-04-21

![v10.0.36 更新概览](/img/changelog/v10.0.36.png)

## 修复
- 对话过程中遇到空流、流式长时间无响应、网关不支持流式返回或临时断网时，系统现在会更稳定地自动重试或切换降级方案，减少消息卡住、报错中断和需要手动重发的情况。
- 当对话因上下文过长或模型服务过载而失败时，系统会更智能地自动调整请求参数或切换可用模型，提高长对话和高峰期请求的成功率。
- 聊天中的重试提示现在会显示真实错误原因，并在恢复成功后自动清理中间提示，让你更清楚发生了什么且不会留下多余系统消息。
- 电脑进入休眠时，正在执行的 Agent 会话会被及时中止，避免唤醒后出现异常状态；仅锁屏时任务仍可继续运行。
- 在“运行环境”页面中，Node.js、npm、pnpm、yarn 和 Python 可用版本加载失败时不再误显示“暂无数据”，网络恢复后会自动重试并重新加载。
- “运行环境”页面首次打开或刷新时，不再短暂把检测中的工具链误显示为红色“异常”，而会明确展示“检测中”“正常”或“未安装”状态。

## 改进
- “运行环境”页面的状态提示更加准确直观，版本数量、工具链徽章和包管理器状态在加载过程中会显示更清晰的等待反馈，减少误解。
- Python 可用版本列表的获取逻辑与其他运行环境保持一致，在异常情况下会持续自动重试，提升版本列表加载的稳定性。
