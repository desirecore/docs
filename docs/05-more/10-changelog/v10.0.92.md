# DesireCore v10.0.92

**发布日期**: 2026-07-20

![v10.0.92 更新概览](/img/changelog/v10.0.92.png)

## 新功能
- **交互式终端系统**：Agent 和你可以共享同一个终端面板协同操作，支持持久会话与 claude、codex、gemini、aider 等 CLI 工具的多终端同时控制。
- **手动多会话模式**：特定 Agent 支持新建、切换、重命名、删除独立会话，像 Claude Code 一样手动管理多段不合并的对话历史。
- **Agent 上下文自控**：Agent 可通过 `ManageContext` 工具主动重置上下文、钉住关键内容、丢弃指定工具结果或调整裁剪预算，让对话更聚焦。
- **委派上下文控制**：委派子 Agent 时，可精确收窄其能力声明、记忆范围和系统提示范围，只给子任务所需的最小上下文。
- **委派过程可观测**：委派卡片新增“查看完整过程”入口，可跳转到子 Agent 会话回看完整执行历史。
- **团队成员会话摘要**：团队面板新增“近期会话摘要”入口，支持查看成员的轮次、工具调用用量以及由 AI 生成的深度摘要。
- **InspectRuns 工具**：Agent 可通过新工具查看其他会话的历史消息和结构化摘要，方便多 Agent 协作时的信息同步。
- **批量删除消息**：多选消息后可通过操作栏一键批量删除，支持跨会话清理。

## 修复
- 修复对话回撤或续聊后继续发消息时，因缺失会话目录导致写入报错、消息丢失的问题。
- 修复应用重启或断线重连后，审批重放偶发出现重复通知或审批卡死的问题。
- 修复心跳投递队列在异常退出后产生损坏文件，导致日志持续报错刷屏的问题；现已改为原子写入并自动清理坏文件。

## 改进
- **审批系统全面升级**：支持 AI 自动决策、AI 仅建议和前台真人窗口三种 AI 审批模式，统一服务端权威结算，后台委派和调度链路可自动快速批准，前台保留真人抢先窗口，断线重连后可恢复待审批卡片和 AI 建议，七种审批偏好拥有完整的可视化界面。
- **安装状态更真实**：安装或卸载 MCP 服务时，界面先显示“安装中/卸载中”中间状态，待 Agent 实际校验成功后才变为最终状态，不再出现假“已安装”提示。
- **内置 Agent 更新至 v1.9.0**：安装管理流程升级为端点回写与强校验，提升安装可靠性。
- 部分模型网关返回的上下文超限提示（无具体数字）现在也能被正确识别，并自动触发降输出或重新裁剪上下文，减少因未识别导致的直接报错。
- 流式请求的重试预算按失败原因独立计算，不同原因的失败不再互相占用量，降低因传输层偶发错误导致的重试耗尽。
- 长任务中短暂的流式失败不会累积到整轮对话的重试限制，仅连续失败达到上限才报错，提高长任务成功率。
