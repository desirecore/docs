# DesireCore v10.0.14

**发布日期**: 2026-03-19

![v10.0.14 更新概览](/img/changelog/v10.0.14.png)

## 新功能
- **消息朗读**：现在可以直接对智能体消息一键朗读，并支持播放与停止切换，在桌面端优先使用系统原生语音、不可用时自动回退到浏览器语音。
- **/new 上下文隔离**：现在输入 `/new` 后可以开启一段真正独立的新对话，并在聊天中看到清晰的“上下文已隔离”分隔提示。
- **新增模型支持**：内测供应商新增了 `GLM-5-Turbo` 和 `MiniMax-M2.7`，同时内置 MiniMax 供应商预设同步加入 `MiniMax-M2.7` 相关模型，方便直接选择使用。

## 修复
- 修复了 `/new` 后智能体仍会带上上一段对话上下文的问题，现在新对话不会再“记得”刚才聊过的内容。
- 修复了切换模型后实际仍可能继续使用旧模型的问题，现在更换模型后会确保按当前所选模型发起对话。
- 修复了发送消息时偶发出现报错导致无法正常继续对话的问题。
- 修复了小上下文窗口模型读取大文件时容易触发 400 错误的问题，并在超出上下文限制时提供更易理解的中文提示。
- 修复了 MiniMax 等模型在部分网关场景下因工具参数不完整而持续返回 400 错误的问题。
- 修复了 Windows 下拖拽文件后消息中显示完整路径、以及智能体偶尔找不到文件的问题。
- 修复了在本地目录中手动新增智能体后，应用内智能体/会话列表不会自动刷新的问题。
- 修复了创建或编辑智能体配置时因 `$schema` 与版本号格式不一致导致校验失败、无法保存的问题。

## 改进
- 文件读取与上下文裁剪策略更加智能，小窗口模型在处理长文本和中文内容时会更稳定、更少出现无效重试。
