# DesireCore v10.0.39

**发布日期**: 2026-04-27

![v10.0.39 更新概览](/img/changelog/v10.0.39.png)

## 新功能
- **云端发布与导入增强**：现在你可以在发布 Agent 时选择公开或私有、为已发布的 Agent 一键复制分享链接或转为公开，并支持直接从公开 URL 导入 Agent。
- **Agent 版本管理**：现在可以在应用内为 Agent 执行版本发布，自动递增补丁版本、生成更新记录并创建版本标签，同时面板会显示本地与远程版本号。
- **媒体生成技能**：新增文生图、文生视频和语音合成等媒体技能，用户可以直接让 Agent 生成图片、视频和语音内容。
- **SuperDoc 智能编辑增强**：在文档中选中文本后可直接发起改写，文档问答中的引用可点击定位到原文位置，并新增 Markdown 结构检查面板帮助快速发现格式问题。
- **Mermaid 图表预览**：SuperDoc 现在支持在编辑器中直接渲染 Mermaid 图表预览。

## 修复
- 长时间未操作后再次发送消息时，不会再出现消息发不出去且界面一直停留在“思考中”的情况。
- 修复了重试或编辑历史消息时可能误删大量聊天记录的问题，历史对话会更安全地保留下来。
- 清除对话历史时会更彻底，不再出现清空后刷新又“复活”旧会话内容的情况。
- 未配置好模型或密钥时报错后，界面中的“加载工具和技能”等状态现在会正确结束，不再一直卡住。
- 点击暂停已结束的会话时，不会再错误提示“会话不存在或已停止”。
- 重试失败消息后，旧的错误提示不会再残留在对话中。
- 修复了多 Agent 并行工作时文档面板和工作流面板互相抢占的问题，不同会话的文档与工作流视图现在会彼此隔离。
- 修复了 AskUserQuestion 问题卡片会漂浮到其他对话页或其他标签页上的问题，现在只会在对应对话中展示并按场景正确提醒。
- 修复了同协议下多个供应商使用同名模型时可能串用错误 API Key 的问题，模型选择会准确对应到你指定的供应商。
- 修复了本地视频无法正常播放的问题，并补齐了本地媒体协议权限，使本地图片、音频、视频的内联播放更稳定。
- 修复了媒体上传对视频支持不完整的问题，现在视频文件也能正常上传处理。
- 修复了审计日志数据量较大时审计页面加载缓慢甚至崩溃的问题。
- 修复了 Kimi 的 thinking 能力兼容问题，相关推理表现已恢复正常。
- 修复了 MiniMax 媒体技能的供应商绑定错误，相关图片、语音和视频技能现在能正确走统一的 MiniMax 配置。

## 改进
- 当 Agent 使用媒体技能时，会更倾向于优先调用与你当前模型供应商一致的能力，减少“选错技能”的情况。
- 媒体内容现在支持更好的消息内联展示与播放，生成后的图片、音频、视频查看体验更顺畅。
- 在 Windows 等缺少 Git Bash 的环境下，媒体技能的使用兼容性更好，跨平台体验更稳定。
- SuperDoc 工具栏与浮动操作区在窄屏或分屏场景下显示更紧凑，不容易出现按钮被挤掉或看不到的问题。
- 文档被 Agent 修改后的差异检测与展示更加稳定，减少了变更丢失或无法正确显示的问题。
- 对话初始化与 `/new` 新会话的历史定位更准确，刷新后更不容易回到错误的旧运行记录。
- 核心智能体 `desirecore` 现在会明确禁止在客户端直接发布，避免误操作。
