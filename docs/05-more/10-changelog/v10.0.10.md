# DesireCore v10.0.10

**发布日期**: 2026-03-13

## 新功能
- **消息排队持久化**：现在可连续发送多条待处理消息，应用重启后也会自动恢复并继续发送，不再因中断而丢失排队内容。
- **SuperDoc 精细审查**：在 SuperDoc 中审查 Agent 修改文件时，现已支持按单个变更块逐段接受或拒绝，并可直接保存文档。
- **Markdown 文件快捷打开**：聊天消息里出现的 `.md` 文件名现在可以直接点击打开，查看和继续编辑更方便。
- **委派感知协作智能体**：Agent 在决定委派任务时，能够感知当前可协作的其他 Agent 及其状态，从而更精准地选择委派对象。
- **独立 Canvas 能力**：Canvas 现已作为独立能力提供，相关调用更清晰，使用体验更稳定。

## 修复
- 修复了多 Agent 并行使用时消息和活动状态可能串到其他对话中的问题，切换 Agent 后显示更准确。
- 修复了非预置 Agent 头像经常显示成同一个蓝色图标的问题，现在不同 Agent 更容易区分，也支持自定义头像。
- 修复了消息复制失败时仍提示“已复制”的误导性反馈。
- 修复了 Agent 正在思考时仍可点击重试或编辑导致异常操作的问题。
- 修复了重试或编辑消息后文件引用可能丢失的问题。
- 修复了重试后模型仍沿用旧上下文、继续生成时异常文本混入聊天记录的问题。
- 修复了流式输出时聊天列表滚动判断不准确的问题，消息浏览与自动滚动更符合预期。
- 修复了技能同步与卸载过程中可能导致版本判断或锁定状态不一致的问题，技能更新更可靠。

## 改进
- **消息操作体验优化**：复制、重试、编辑按钮改为悬停时显示并与时间戳同排展示，聊天界面更清爽。
- **长消息编辑优化**：短消息可直接原位编辑，长消息则会弹出更大的编辑窗口，修改内容更顺手。
- 流式回复过程中现在可以自由上滑查看历史消息，界面不会强行拉回到底部。
- 停止生成按钮已独立于发送按钮，AI 回复中也可以继续发送新消息。
- Agent 通过 delegate 委派任务时，子 Agent 现在能更完整地使用技能、工具与上下文能力，委派结果更稳定。
- 连接 HostAgent 时会优先显示你的系统主机名作为设备名称，远端识别设备更直观。
- 设备网络中的本机识别与展示方式得到优化，本机设备更容易辨认，设备列表也更统一。
