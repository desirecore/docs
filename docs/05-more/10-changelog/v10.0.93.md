# DesireCore v10.0.93

**发布日期**: 2026-07-21

![v10.0.93 更新概览](/img/changelog/v10.0.93.png)

## 新功能
- **终端主题实时跟随**：终端配色现在会随系统亮色/暗色模式自动切换，无需重新打开终端。
- **终端分屏布局**：终端面板新增 Split 分屏模式，终端占满左侧半屏、对话区收窄到右侧，与底部浮层 Dock 模式可一键切换。
- **终端数量提升与选择器**：单个 Agent 终端上限从 5 个提升至 1000 个；超过 12 个终端时标签栏和状态条自动收起为「+N」入口，点击可弹出搜索选择器快速跳转。
- **终端运行状态条**：输入框下方新增紧凑的终端运行状态条，Agent 后台有终端运行时直接显示芯片，点击即可打开终端面板。
- **图片内置预览**：消息中的图片附件和缩略图支持应用内分屏预览，支持缩放、平移、适配和沉浸模式（Alt 点击直接沉浸）。
- **输入框配置胶囊**：输入框新增审批模式胶囊（左侧）和模型/思考深度合并胶囊（右侧），无需打开资源面板即可快速切换审批档位和模型参数。
- **输入框两行布局**：输入区独占全宽，工具栏（附件、截图、胶囊、发送按钮等）分布在下行，大幅提升输入空间。
- **并行多实例**：开发模式通过 `--home` 参数可一键启动隔离的并行实例；生产环境重复启动时弹出实例选择器，支持新建或复用历史数据目录。
- **团队导出全覆盖**：团队导出现在包含团队共享记忆、共享技能、团队提示词、成员声明和工作目录等完整子树，不再仅打包 `team.json`。
- **按成员查看会话**：`InspectRuns` 工具现在支持直接传成员名称或 ID 定位其最新会话，与 Delegate 工具的成员解析规则完全一致。

## 修复
- **重试态卡死**：手动多会话模式下，后台委派会话完成后「重试中」提示不再卡死，complete 事件按会话归属精确匹配。
- **启动灰屏**：修复重度使用场景下，巨型运行记录文件（实测 988MB）导致启动时 V8 内存溢出、客户端呈现灰屏的严重问题，改为有界读取。
- **模型误报错误**：修复首次启动时输入框模型胶囊偶发误报「提供商已移除，当前模型不可用」的竞态问题。
- **菜单截断**：修复聊天头部「…」菜单在条目过多（如团队模式 20 项）时超出窗口底部被截断的问题，改为两列布局。

## 改进
- **thinking 降级重试**：thinking 模式下工具调用违规时，不再原样重放注定失败的请求；首次违规给一次重试机会，第二次连续命中同一违规时自动降级思考模式并继续执行。
- **菜单背景可读性**：输入框审批模式和模型/思考深度胶囊的弹出菜单改为不透明背景，内容不再被下层文字透出干扰。
- **手动会话侧栏打磨**：手动多会话模式侧栏新增加载骨架屏、错误提示和进出场动画，删除操作改用自定义确认弹窗而非浏览器原生弹窗。
- **终端协议瘦身**：终端列表不再附带每个终端最多 500KB 的缓冲区内容，改为按需拉取，避免大量终端场景下传输数百 MB 数据。
