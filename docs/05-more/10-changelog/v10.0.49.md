# DesireCore v10.0.49

**发布日期**: 2026-05-08

![v10.0.49 更新概览](/img/changelog/v10.0.49.png)

## 新功能
- **平台定向更新检测**：客户端现在会按你当前使用的操作系统单独获取可用版本，让新版本在对应平台构建完成后更快出现在更新提示中。
- **通义千问算力与编码方案扩展**：新增和完善了 DashScope 相关模型提供商、计算预设与编码方案，方便你在创建或使用智能体时选择更多可用算力配置。

## 修复
- 智能体自动更新在网络异常、仓库无权限或远端信息获取失败时，不再误报“已是最新版本”，而会明确提示你检查网络或远程仓库状态。
- 手动点击“检测更新”时会直接获取最新远端版本信息，修复了因缓存导致刚发布的新版本暂时检测不到的问题。
- 智能体自动更新遇到 Git 冲突时，现在会明确显示冲突文件，并恢复在聊天头部显示可持续保留的“冲突”标记，方便你定位和处理问题。
- 自动更新失败时会根据具体原因给出更准确的提示，例如未配置远程仓库、配置无效、当前目录不是 Git 仓库等，不再统一显示模糊的失败信息。
- 新建智能体后其 Git 仓库初始化更加稳定，修复了刚创建完成就执行 Git 操作时偶发失败的问题。

## 改进
- 智能体名称旁的冲突状态支持跨重启保留并自动刷新，只要冲突仍存在，你重新打开客户端后也能继续看到提醒。
- 自动更新拉取代码时会优先使用智能体配置中的目标分支，减少因分支不一致导致的更新异常。
- 工作流相关默认技能与模板配置已更新，默认体验更贴近当前产品能力。
