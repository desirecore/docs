# DesireCore v10.0.33

**发布日期**: 2026-04-16

![v10.0.33 更新概览](/img/changelog/v10.0.33.png)

## 新功能
- **工作流系统**：新增可视化工作流功能，你可以在资源管理器中创建、查看和编辑工作流，并通过画布编排触发、代码、模型、智能体和人工审批等节点完成复杂任务。
- **工作流运行与追踪**：工作流现已支持手动运行、Webhook 触发、执行中取消、实时进度推送和执行历史查看，让你更容易追踪每次运行结果。
- **工作流结果卡片**：智能体在对话中执行工作流后，会直接以结构化结果卡片展示状态、进度、输出和错误信息，查看结果更直观。
- **PowerShell 工具**：Windows 用户现在可以让智能体直接通过内置 PowerShell 执行系统命令，更适合本机脚本、系统管理和命令行任务。
- **工作流 Skill 增强**：新增工作流相关内置能力，智能体可以更完整地帮助你校验、测试、运行或直接打开工作流文件。
- **Skill 元信息展示**：技能列表现在可以正确显示技能作者和版本信息，方便你识别技能来源和更新情况。

## 修复
- 修复了同时调用多个 Write 工具时只显示最后一个文件名的问题，现在批量写入多个文件时会正确显示对应文件。
- 修复了多个同名工具连续执行时摘要分配错位的问题，现在每个工具的执行摘要都会准确对应到自己的结果。
- 修复了刷新后用户消息中可能暴露 Write/Edit 提示指令的问题，聊天记录展示更干净也更安全。
- 修复了团队工作区路径被错误分类导致导出失败的问题，现在取消勾选工作区文件后不会再误导出大量团队工作区内容。
- 修复了工作流中部分模型选择、节点输入、空响应和错误原因展示不准确的问题，运行失败时能看到更明确的反馈。
- 修复了工作流编辑面板输入时容易失焦、代码编辑器在部分场景下不显示或高度异常的问题，编辑体验更稳定。
- 修复了 Windows 下 PowerShell 只读命令仍触发审批、命令无输出以及中文乱码等问题，命令执行结果更符合预期。

## 改进
- ProcessGroup 现在会在空间不足时自动优化排版，并在文件较多时以更清晰的方式展示文件名和数量。
- 工作流编辑器整体交互与界面表现得到提升，包括节点图标、表单样式、状态提示、暗色模式适配、配置/运行结果切换和历史面板体验。
- 工作流中的 LLM 节点配置更加完善，支持更清晰的模型选择、提示词引用、温度调节、输出结构配置和代码编辑。
- 工作流编辑支持自动保存与并发编辑保护，你在修改节点配置时内容会更平滑地保存，减少覆盖和冲突风险。
- 数据导出流程新增更明确的导出进度与结果提示，让你能更直观地确认导出是否成功。
