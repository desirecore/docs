# DesireCore v10.0.43

**发布日期**: 2026-05-01

![v10.0.43 更新概览](/img/changelog/v10.0.43.png)

## 新功能
- **全新更新流程**：应用发现新版本时会先展示更新说明，并提供“立即更新、后台更新、下次自动更新、跳过该版本、稍后”五种选择，让你更可控地决定何时升级。
- **自动更新模式**：你现在可以为当前更新通道开启自动更新，后续新版本会自动在后台下载，并在准备完成后通过倒计时提醒你重启安装。
- **draw.io 图表编辑支持**：现在可以在智能体文件编辑器中直接插入和编辑 draw.io 图表，并在文档中实时预览渲染结果。
- **会话日期感知**：智能体在长时间对话中会自动感知当前本地日期与跨日变化，减少“今天/明天”等时间相关回答出错的情况。

## 修复
- 修复了使用部分 Anthropic 兼容网关时，对话偶发因流式事件顺序异常而中断的问题，整体对话稳定性更好。
- 修复了上下文超限时在多种模型和网关下容易直接失败的问题，现在系统会更智能地降级重试并尽量保留已生成内容。
- 修复了智能体调用编辑工具自动打开 SuperDoc 分屏时，输入框正在编辑的文字、图片附件和输入状态可能被清空的问题。
- 修复了在不同会话之间切换后，输入框里已上传完成的图片附件会丢失的问题。
- 修复了暂停生成后界面仍错误显示智能体“工作中”的问题，状态显示与“继续生成”按钮现已保持一致。
- 修复了创建团队时引用不存在的成员智能体仍能成功创建、进而导致团队工作目录绑定异常的问题。
- 修复了在文档中插入 draw.io 图表时可能插到开头而不是光标位置、且插入后不能立即渲染的问题。
- 修复了 MiniMax 等媒体能力相关配置下的若干问题，包括空媒体文件播放异常、空文件上传异常以及部分供应商详情页可能报错的问题。
- 修复了同一 slug 下存在多个供应商时，媒体技能可能优先命中未配置密钥的供应商而无法正常使用的问题。

## 改进
- 算力服务配置体验得到增强，媒体模型现在可以进行更准确的可用性校验，并会在 MiniMax API Key 与 Token Plan Key 配错位置时主动提醒你。
- MiniMax 相关模型与套餐预设已更新，媒体技能默认模型与可选模型更完整，使用 Token Plan 时的命名也更清晰统一。
- 内测通道新增小米 MiMo V2.5 与 MiMo V2.5 Pro 模型，可在支持的环境中直接选择使用。
