# DesireCore v10.0.73

**发布日期**: 2026-06-05

![v10.0.73 更新概览](/img/changelog/v10.0.73.png)

## 新功能
- **离线图表编辑与对话画图**：现在你可以在完全离线的情况下使用 draw.io 进行专业图表编辑，并让 Agent 在对话中直接生成可继续编辑的流程图、架构图等图表卡片。
- **运行环境注入到 Agent 上下文**：Agent 现在能感知当前操作系统、可用工具链以及 DesireCore 内置的 Node/Python 运行时信息，从而更准确地生成和执行任务方案。
- **外部服务凭据安全授权**：你现在可以让 Agent 访问 HTTP、MCP、脚本和 Bash 等外部能力时通过占位符安全授权，避免明文凭据暴露给 Agent、模型上下文或日志。
- **Office 能力离线依赖预装**：Office 类技能所需依赖现在会随客户端预装并跨平台复用，使相关技能首次使用更快、离线也可用且无需额外安装环境。

## 修复
- 修复了部分情况下添加模型提供商后 `compute.json` 被写坏、导致模型列表清空且只能重装的问题，配置异常时现在会自动校验、备份并尽可能恢复。
- 修复了 Kimi 在 thinking 模式下进行多轮工具调用时，可能因缺少推理内容字段而直接报错的问题。
- 修复了聊天历史持久化后 draw.io 图表卡片可能退化成普通“执行了工具”提示、重启后无法正确重渲染的问题。

## 改进
- web-access 浏览器固定连接在目标浏览器关闭或失联后现在可自动恢复，无需重启代理即可重新发现可用浏览器。
- 运行环境页面手动刷新后现在会明确提示刷新成功或失败，避免你误以为操作没有生效。
- 本次内置技能市场同步了多项全局技能版本，已升级用户也能正常收到这些技能内容更新。
- 应用底层运行时升级后，整体稳定性与启动性能有所提升。
