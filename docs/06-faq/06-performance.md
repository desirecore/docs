# 性能问题

## 应用占用过多内存？

DesireCore 基于 Electron 构建，会占用一定的基础内存。如果内存占用异常偏高：

1. **检查对话数量** --- 过多的对话窗口会增加内存占用。关闭不需要的对话或清除长时间不使用的对话历史
2. **清理图片缓存** --- 对话中上传的图片会缓存在内存中，大量图片会显著增加内存使用
3. **重启应用** --- 长时间运行后内存可能逐渐增长，定期重启可以释放累积的内存
4. **检查 Companion 数量** --- 同时运行多个 Companion 会增加内存开销

**参考内存占用：**

| 场景 | 预期内存占用 |
|------|------------|
| 刚启动 | 200-400 MB |
| 正常使用（2-3 个对话） | 400-800 MB |
| 大量对话 + 图片 | 800 MB - 1.5 GB |

## 界面卡顿或动画不流畅？

DesireCore 使用 Liquid Glass 设计语言，依赖 GPU 加速的模糊和透明效果。如果界面不流畅：

1. **更新显卡驱动** --- 确保你的显卡驱动是最新版本
2. **检查 GPU 使用率** --- 如果其他应用大量占用 GPU，DesireCore 的渲染性能可能受影响
3. **减少背景应用** --- 关闭其他 GPU 密集型应用（如视频编辑、游戏）
4. **降低系统分辨率** --- 高分辨率显示器上渲染开销更大

:::tip
macOS 上 DesireCore 会使用系统原生的窗口模糊效果（vibrancy），通常性能较好。Windows 和 Linux 上使用 CSS `backdrop-filter`，对 GPU 要求稍高。
:::

## AI 回复的响应速度慢？

响应速度主要取决于 AI 模型 API 的性能，与 DesireCore 客户端本身关系较小。优化建议：

1. **选择更快的模型** --- 轻量模型（如 Claude Haiku、GPT-4o-mini）响应速度明显快于旗舰模型
2. **缩短上下文** --- 对话越长，每次请求需要处理的 Token 越多。适时清除上下文
3. **使用就近的 Provider** --- 选择地理位置较近的 API 端点
4. **检查网络质量** --- 使用稳定的网络连接，减少丢包和延迟

## 如何查看应用的运行日志？

如果需要排查性能问题，可以查看应用日志：

- **macOS**: `~/Library/Logs/desirecore/`
- **Windows**: `%APPDATA%/desirecore/logs/`
- **Linux**: `~/.config/desirecore/logs/`

开发者工具中也可以查看渲染进程的控制台输出（在应用中按 `Ctrl/Cmd + Shift + I`）。

## 大文件处理时应用无响应？

如果向 Companion 发送或引用大文件时应用无响应：

1. **避免发送过大的文件** --- 建议单个文件不超过 10 MB
2. **分批处理** --- 大型任务拆分为多个小任务
3. **使用文件引用而非直接上传** --- 引用路径让 Companion 按需读取，比一次性上传更高效

## 应用更新后性能变差？

如果更新后明显感觉性能下降：

1. **清除应用缓存** --- 删除应用缓存目录
2. **重新登录** --- 退出后重新启动
3. **检查更新日志** --- 了解新版本的变化，是否有已知的性能问题
4. **反馈问题** --- 通过反馈渠道报告，帮助开发团队定位问题
