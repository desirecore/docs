# 数据科学家

## 场景描述

你是产品团队的负责人，最近用户流失率上升了，但你不确定原因。手上有大量的用户行为数据，需要有人帮你从中找出线索，建立预测模型，并给出可操作的改进建议。

数据分析师智能体不仅能做基础的统计分析，还能处理复杂的用户行为分析、建立预测模型、搭建运营监控仪表盘，帮你从数据中发现业务机会。

## 推荐智能体

**数据分析师** -- 支持高级统计分析、预测模型、用户分群、异常检测和实时监控仪表盘。

## 完整对话示例

```
你: 分析我们 App 的用户行为数据，找出用户流失的原因。
    [上传 user_behavior.json]

数据分析师: 正在进行用户行为分析...

        用户概况
        - 总用户数：50,000
        - 月活用户（MAU）：32,000（64%）
        - 本月流失用户：3,200（6.4%）

        用户旅程漏斗
        注册 → 首次使用 → 激活 → 留存 → 付费
        100%    78%       45%    32%    8%

        关键流失节点
        注册 → 首次使用：流失 22%（行业平均 15%）
        首次使用 → 激活：流失 42%（行业平均 35%）

        流失原因分析
        1. 新手引导流程过长（35% 用户提及）
           平均完成时间 8 分钟，建议缩短至 3-5 分钟
        2. 核心功能入口不明显（28% 用户提及）
           68% 用户在首页停留超 30 秒未操作
        3. 首次使用无价值感（22% 用户提及）
           72% 未体验到核心功能即退出

        用户分群
        高价值用户（8%）：日均使用 30+ 分钟
        潜力用户（24%）：每周 3+ 次但未付费
        流失风险用户（18%）：连续 7 天未登录

        预测
        不采取措施：下月流失率上升至 8.5%
        优化新手引导：预计降低流失率 2-3%

你: 能给出一个优先级排序的改进方案吗？

数据分析师: 改进优先级排序（按预期效果）：
        P0：缩短新手引导流程（影响面最大）
        P1：首页增加核心功能引导
        P2：设计首次使用奖励机制
        P3：对流失风险用户推送召回内容
        ...
```

## 关键步骤

1. **上传行为数据** -- 提供用户行为日志、事件数据等
2. **漏斗分析** -- 智能体自动识别用户旅程中的关键流失节点
3. **深入归因** -- 针对流失节点分析具体原因
4. **预测建模** -- 基于历史数据预测未来趋势
5. **改进建议** -- 按优先级给出可操作的改进方案

## 最终成果

一份完整的用户行为分析报告，包含漏斗分析、流失归因、用户分群、预测模型和优先级排序的改进方案。

:::tip 最佳实践
- 数据越完整，分析越准确。最好包含用户行为事件、时间戳、设备信息等维度
- 先做全局分析了解整体情况，再针对具体问题深入。避免一上来就钻进细节
- 分析结果要落地为可执行的产品改进项，可以让智能体按"投入产出比"排优先级
:::
