# 产品经理

## 场景描述

你正在管理一个产品的迭代开发。本周需要写需求文档、跟踪 Sprint 进度、协调下周的资源分配，还要准备一个跨部门的上线协调会。每件事都不难，但全部加起来让你分身乏术。

产品经理（Product Manager）智能体可以协助你完成这些日常工作：撰写结构化的需求文档、实时跟踪 Sprint 进度、合理分配开发资源、识别项目风险。

## 推荐智能体

**产品经理** -- 专精于项目管理与协调，擅长进度追踪、资源分配、风险管控和团队协调。

## 完整对话示例

```
你: Sprint-7 进度怎么样了？

产品经理: Sprint-7 进度报告

        周期：6月3日 至 6月14日（第 6 个工作日）
        总体进度：85%（时间进度 60%，进度超前）

        任务完成情况
        - 已完成：18/24 个（75%）
        - 进行中：4 个
        - 待开始：2 个

        各成员进度
        张三（前端）：已全部完成 6/6
        李四（后端）：进行中 4/5，剩余"认证接口优化"
        王五（后端）：进行中 3/5，"订单模块重构"预计 2 天
        赵六（测试）：完成 4/6，集成测试等待开发完成

        风险预警
        - 订单模块重构进度略慢，建议张三协助王五
        - 集成测试依赖后端任务，需优先保障

        预计完成率：95%
        建议：将 1 个低优先级任务移至下个 Sprint

你: 帮我写一份"用户标签系统"的需求文档。

产品经理: 用户标签系统 -- 需求文档

        一、需求背景
        运营团队需要对用户进行精细化分群，目前只能通过
        手动筛选，效率低且无法实时更新...

        二、核心功能
        1. 标签创建与管理
        2. 自动标签规则引擎
        3. 用户分群查询
        4. 标签数据看板

        三、用户故事
        作为运营人员，我希望能够创建自定义标签并设定
        自动打标规则，以便自动化用户分群...

        四、验收标准
        ...
```

## 关键步骤

1. **查看迭代进度** -- 随时询问当前 Sprint 的完成情况和风险
2. **撰写需求文档** -- 提供功能描述，智能体生成结构化 PRD
3. **资源调度** -- 分析多项目资源冲突，给出分配建议
4. **风险管理** -- 自动识别项目风险，提出应对措施

## 最终成果

高效的迭代管理支持，包括进度报告、需求文档、资源分配方案和风险预警。

:::tip 最佳实践
- 每日站会前让智能体生成进度摘要，快速了解团队状态
- 写需求文档时先描述业务场景和用户痛点，智能体会自动拆解为用户故事和验收标准
- 多项目并行时，让智能体分析资源冲突并推荐最优分配方案
:::
