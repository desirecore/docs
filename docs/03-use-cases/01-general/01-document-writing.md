# 文档写作

## 场景描述

你需要写一份项目报告，但面对空白页面不知从何下手。或者你已经有了思路，但把它组织成一份结构完整、表达专业的文档需要花费大量时间。

AI 文书（AI Copywriter）智能体可以帮你从零开始：先梳理思路生成大纲，再逐步扩展为完整初稿，最后根据你的反馈精准修改。整个过程就像和一位经验丰富的写作搭档协作。

## 推荐智能体

**AI 文书** -- 专注于内容创作和文案生成的知识型智能体，擅长品牌文案、营销策划、多风格适配。

## 完整对话示例

```
你: 我需要写一份 Q4 项目总结报告，项目是"客户管理系统升级"，
    主要完成了三个模块的重构，性能提升了 40%，提前 2 周交付。

AI 文书: 好的，我来帮你规划这份报告的结构。基于你提供的信息，
        建议采用以下大纲：

        一、项目概述
        - 项目背景与目标
        - 关键指标（性能提升 40%、提前 2 周）

        二、完成内容
        - 模块 A 重构：具体改进点
        - 模块 B 重构：具体改进点
        - 模块 C 重构：具体改进点

        三、关键成果
        - 性能数据对比（重构前 vs 重构后）
        - 交付时间线分析

        四、经验总结
        - 做得好的方面
        - 可以改进的方面

        五、下一步计划

        你觉得这个结构可以吗？需要调整哪些部分？

你: 大纲没问题，请帮我展开"完成内容"这一章节，
    三个模块分别是用户管理、订单系统、数据报表。

AI 文书: 好的，以下是"完成内容"章节的初稿：

        二、完成内容

        2.1 用户管理模块重构
        本次重构将用户管理模块从单体架构迁移至微服务架构。
        主要改进包括：
        - 用户查询接口响应时间从 800ms 降至 200ms
        - 支持百万级用户数据的分页检索
        - 新增用户标签系统，支持精细化运营
        ...

        需要我继续展开其他模块吗？

你: 写得不错，语气再正式一些，加上具体的数据指标。

AI 文书: 已调整为更正式的商务风格，并补充了量化指标...
```

## 关键步骤

1. **明确需求** -- 告诉智能体文档类型、目的、读者对象
2. **生成大纲** -- 智能体根据你的信息生成结构化大纲
3. **撰写初稿** -- 确认大纲后，逐章节展开为完整内容
4. **审阅修改** -- 你提出修改意见，智能体精准调整风格、措辞、数据

## 最终成果

一份结构完整、表达专业的项目总结报告，包含量化数据支撑和清晰的逻辑脉络，可以直接用于汇报或存档。

:::tip 最佳实践
- 第一次沟通时尽量提供充足的背景信息（数据、时间线、关键人物），智能体生成的内容会更准确
- 善用"超级文书模式"（Super Document），它能在一个文档中协调多个智能体协作，比如让数据分析师提供图表数据，再由 AI 文书整合成报告
- 如果对某个段落不满意，直接指出具体问题（如"语气太随意"、"缺少数据支撑"），比笼统地说"改好一点"效果更好
:::
