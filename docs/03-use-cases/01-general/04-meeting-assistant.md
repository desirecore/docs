# 会议助手

## 场景描述

你下午有一个重要的项目协调会，需要准备议程。会后还要整理会议纪要、提取每个人的待办事项、发给所有参会人。这些事务性工作虽然不难，但非常耗时。

AI 助手可以帮你高效完成会议的前前后后：会前生成结构化议程，会后根据你的笔记或录音转写文稿整理出正式纪要，并自动提取待办事项分配到具体负责人。

## 完整对话示例

```
你: 明天下午有个产品上线协调会，参会人有产品经理、开发负责人、
    测试经理和运维经理。主要讨论上线时间和灰度方案。
    帮我准备一份议程。

AI 助手: 好的，以下是会议议程草案：

        产品上线协调会
        时间：明天下午 14:00-15:30
        地点：[待确认]

        议程安排：
        14:00-14:10  开场：项目进展概述（产品经理）
        14:10-14:30  开发完成情况及遗留问题（开发负责人）
        14:30-14:50  测试进展与质量报告（测试经理）
        14:50-15:10  上线方案与灰度策略（运维经理）
        15:10-15:25  讨论：确认上线时间和回滚方案
        15:25-15:30  总结：明确各方待办和截止时间

        待决策事项：
        1. 上线时间选择（周五下午 or 下周一上午）
        2. 灰度比例（10% → 50% → 100%）
        3. 回滚触发条件

        需要调整吗？

你: 议程没问题。会开完了，我把记的笔记发给你，帮我整理纪要。
    "决定下周一上线，先灰度 10%，观察 2 小时没问题再扩到 50%，
    隔天全量。开发这边还有 2 个 P1 的 bug 要周五前修完，
    测试周末加班做回归。运维准备回滚脚本，周日前完成。"

AI 助手: 会议纪要已整理完毕：

        会议纪要
        会议主题：产品上线协调会
        日期：[今天日期]

        一、决议事项
        1. 上线时间：下周一上午
        2. 灰度策略：10% → 观察 2 小时 → 50% → 隔天 100%
        3. 回滚方案：运维准备自动回滚脚本

        二、待办事项
        | 事项 | 负责人 | 截止时间 |
        |------|--------|---------|
        | 修复 2 个 P1 Bug | 开发负责人 | 周五 |
        | 回归测试 | 测试经理 | 周末 |
        | 准备回滚脚本 | 运维经理 | 周日 |

        三、下次跟进
        周一上午 9:00 上线前最终确认

        可以直接发送给参会人吗？
```

## 关键步骤

1. **会议议程生成** -- 提供参会人和讨论主题，智能体生成结构化议程
2. **会议记录整理** -- 会后提供笔记或转写文稿，智能体整理为正式纪要
3. **待办事项提取** -- 自动从纪要中提取行动项，标注负责人和截止时间

## 最终成果

一份可直接发送的正式会议纪要，包含决议事项、待办清单和下次跟进安排。

:::tip 最佳实践
- 会议笔记不需要完整记录每句话，只需记下关键决定和分工即可，智能体会帮你组织成完整纪要
- 如果有会议录音转写文稿，直接粘贴给智能体效果更好
- 纪要格式可以让智能体记住你的偏好（比如"每次都用表格列出待办"），下次就不用重复说明了
:::
