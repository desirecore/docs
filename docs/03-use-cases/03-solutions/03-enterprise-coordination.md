# 企业服务协调

## 场景描述

你的公司要在下个月发布新产品。这涉及法务审查产品文案、市场部制作宣传材料、财务部核算成本定价、项目经理制定发布时间表......传统方式需要你逐一联系各部门、催进度、整合信息，协调成本极高。

DesireCore 的核心调度智能体就像一个项目总管，它能理解你的整体需求，自动将任务拆解并委派给最合适的专业智能体并行执行，实时监控进度，最后整合所有成果交付给你。

## 涉及智能体

- **DesireCore 核心** -- 任务编排、智能体调度、全局监控
- **法律顾问** -- 合规审查
- **AI 文书** -- 营销材料撰写
- **财务助手** -- 成本核算和定价
- **产品经理** -- 发布时间表和任务分配

## 完整对话示例

### 一键启动多部门协作

```
你: 我们要在下个月发布新产品，需要准备发布相关的所有材料。

DesireCore: 我来帮你协调这次产品发布。我将它拆解为以下工作：

        1. 法律合规审查 -- 检查产品文案、用户协议
        2. 营销材料准备 -- 产品介绍、宣传文案、社媒内容
        3. 财务成本核算 -- 发布成本、定价策略
        4. 项目规划 -- 发布时间表、任务分配

        已委派各智能体开始工作：
        → 法律顾问：审查产品合规性
        → AI 文书：准备营销材料框架
        → 财务助手：进行成本预估
        → 产品经理：制定发布计划

        预计 30 分钟内返回初步方案。
```

### 实时进度监控

```
你: 现在进展怎么样了？

DesireCore: 各项工作进展如下：

        法律合规审查：已完成
        - 产品文案无合规问题
        - 用户协议需补充数据处理条款

        营销材料：进行中（70%）
        - 产品介绍初稿已完成
        - 社媒内容矩阵制作中

        财务核算：已完成
        - 发布预算：85 万
        - 建议定价区间：299-399 元

        发布计划：已完成
        - 关键里程碑已标注
        - 资源分配无冲突

        需要你确认的事项：
        1. 用户协议数据处理条款（法律顾问已草拟）
        2. 定价选择 299 还是 399（财务助手已给出分析）
```

### 紧急任务插队

```
你: 紧急！重要客户投诉了产品质量问题，需要立即处理。

DesireCore: 收到紧急任务，正在调整资源优先级...

        当前状态：
        - AI 文书：正在制作社媒内容（可暂停）
        - 法律顾问：空闲
        - 客服中心：正在处理 2 个普通咨询

        调度决策：
        → 暂停 AI 文书的社媒任务（已保存进度）
        → 委派客服中心全力处理投诉
        → 法律顾问准备可能的应对方案

        预计 5 分钟内首次回复客户。
```

## 核心能力

### 任务编排

DesireCore 能将一个复杂需求自动拆解为多个子任务，识别任务间的依赖关系，安排串行或并行执行：

- 无依赖的任务并行执行，缩短总耗时
- 有前后依赖的任务按顺序串联
- 自动监控每个任务的完成状态

### 智能调度

当系统资源紧张或出现意外情况时，DesireCore 会：

- 根据任务优先级重新分配资源
- 紧急任务自动抢占低优先级任务的资源
- 某个智能体故障时，自动将任务转派给备用智能体

### 意图识别

DesireCore 能区分你是在**教学（Teach）**还是**委派任务（Delegate）**：

- "以后写邮件用轻松语气" → 教学，记录偏好并同步给所有文案类智能体
- "帮我写封邮件给客户" → 委派，分配给 AI 文书执行

## 最终成果

一个高效运转的多智能体协作系统，将原本需要数天的跨部门协调压缩到几十分钟，所有专业工作并行推进、交叉审核，最终整合为统一的交付成果。

:::tip 最佳实践
- 描述需求时说清全局目标（如"发布新产品"），让 DesireCore 自行判断需要哪些智能体参与，比你逐个指定更高效
- 对于有明确截止时间的项目，一开始就告知时间约束，调度器会优先保障关键路径上的任务
- 定期检查进度汇总，及时对需要你决策的事项做出回应，避免阻塞其他智能体的后续工作
:::
