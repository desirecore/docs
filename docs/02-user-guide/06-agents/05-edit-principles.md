# 编辑行为准则

行为准则（Principles）是智能体的"行为手册"——它规定了智能体必须遵守的规则、禁止触碰的红线，以及决策时的优先级排序。

## 什么是行为准则

行为准则保存在智能体目录下的 `principles.md` 文件中。与人格设定（Persona）不同，准则关注的是"做什么和不做什么"而非"怎么做"。

例如：

| 人格设定（Persona） | 行为准则（Principles） |
|---|---|
| "回答要简洁" | "不得泄露用户隐私信息" |
| "语气要友好" | "涉及付费操作前必须确认" |
| "先给结论再给理由" | "安全 > 正确性 > 效率" |

## L0/L1/L2 内容层级

DesireCore 使用 L0/L1/L2 控制提示词的加载粒度。它们是**内容层级**，不是系统权限等级：

### L0 — 核心摘要

L0 用一句话或极少量内容概括最重要的角色与规则，始终注入系统提示词。

- 不得帮助用户从事违法活动
- 不得泄露其他用户的隐私数据
- 不得伪造数据或凭证

### L1 — 关键行为规范

L1 保存执行任务时需要直接看到的关键要点，默认注入系统提示词：

- 执行删除、发送、付费等操作前必须获得用户确认
- 涉及敏感数据时告知用户风险
- 给出建议时标注不确定性

### L2 — 完整详情

L2 保存较长的解释、例外、示例和操作细节，默认不注入，由智能体在需要时读取：

- 输出格式偏好（Markdown / 纯文本）
- 回答详细程度（简洁 / 详尽）
- 特定领域的工作习惯

| 层级 | 默认加载 | 建议内容 | 可编辑 |
|---|---|---|---|
| **L0** | 始终注入 | 一句话核心摘要 | 是 |
| **L1** | 默认注入 | 关键规则与操作要点 | 是 |
| **L2** | 按需读取 | 完整解释、示例和例外 | 是 |

:::warning 提示词不等于权限
把一条规则写进 L0 不会授予工具、文件、命令或外部服务权限，也不能覆盖 DesireCore 的代码级安全策略和审批规则。反过来，用户可编辑的 L0 也不应被描述为不可修改的系统安全底线。
:::

## 添加和修改准则

### 通过界面编辑

1. 打开 **资源管理器 → 提示词中心**
2. 展开目标智能体，选择“行为准则”
3. 直接编辑 `principles.md` Markdown
4. 保存后打开“提示词预览”，检查它与全局、团队和人格提示词的组合结果

### 通过对话教学

在与智能体的对话中，你可以直接设定准则：

> "以后回复任何投资建议时，都要加上风险提示。"

> "不要主动发送邮件，每次都要先问我。"

智能体会识别这些为行为准则，并在审阅后写入 `principles.md`。

### 直接编辑文件

高级用户可以直接编辑 DesireCore 数据目录下对应智能体的 `principles.md`。文件使用 Markdown；分层时必须使用严格的 `## L0`、`## L1`、`## L2` 二级标题。

## 与其他提示词层的关系

运行时按 **全局 → 所属团队 → Agent 人格 → Agent 行为准则** 组合可编辑提示词，再加入团队身份、工具、记忆等系统上下文。L0/L1/L2 决定每个文件加载多少内容；全局/团队/Agent 决定规则的适用范围。请避免在不同层重复写相互矛盾的规则，并用提示词预览检查最终组合。

:::tip 准则写作建议
- **具体胜于模糊**："涉及金额超过 1000 元的操作前必须确认"比"大额操作要确认"更好
- **可测试**：好的准则应该能明确判断"是否违反"
- **不重叠**：避免在准则和人格设定中重复定义同样的内容
:::
