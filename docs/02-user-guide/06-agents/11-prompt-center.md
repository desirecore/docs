# 提示词中心与三层提示词

提示词中心把分散在用户、团队和智能体目录中的系统提示词放到同一个界面中管理。打开 **资源管理器 → 提示词中心**，左侧选择作用范围，右侧编辑内容或预览组合结果。

## 三个作用范围

| 层 | 影响范围 | 内容 |
|---|---|---|
| **全局** | 当前用户的所有智能体 | 通用偏好、工作方式和跨智能体规则 |
| **团队** | 该团队的全部成员和组长；属于多个团队时可出现多段 | 团队协作约定、共享术语和交付标准 |
| **Agent** | 单个智能体 | `persona.md` 人格和 `principles.md` 行为准则 |

运行时按 **全局 → 团队 → Agent 人格 → Agent 行为准则** 组合这些可编辑片段，然后加入团队身份、工具、Skill、记忆和会话等运行时上下文。团队提示词不能伪造或改写真实团队身份；成员、组长和同伴关系来自团队配置。

## L0 / L1 / L2

全局和团队编辑器支持表单与 Markdown 源码两种模式。分层文件必须使用严格的二级标题：

```md
## L0
一句话核心摘要

## L1
- 每次任务都需要看到的关键要点

## L2
较长的背景、示例、例外和操作细节
```

| 层级 | 运行时行为 |
|---|---|
| **L0** | 始终注入系统提示词 |
| **L1** | 默认注入系统提示词 |
| **L2** | 默认不注入，保留为按需读取的详情 |

L0/L1/L2 表示加载粒度，不是权限或安全等级。保存全局、团队或 Agent 文件不会自动扩大工具权限、文件范围或审批权限。

如果源码没有识别到分层标题，DesireCore 会把整份文件作为一个提示词片段注入，并在校验或保存时给出格式提示。

## 编辑和预览

1. 选择“全局提示词”、某个团队，或展开某个智能体
2. 在表单模式填写 L0/L1/L2，或在源码模式编辑完整 Markdown
3. 保存内容
4. 对 Agent 选择“提示词预览”，按全局、团队、Agent 和系统分组检查组合结果

预览会显示片段来源和字符数，适合发现重复、冲突和过长内容。预览反映查询时可加载的静态上下文；实际请求还可能动态加入 inbox、handoff、语音、工具或最新团队状态，因此排查线上请求时应以接口审计记录的系统提示词摘要为最终依据。

## 写作建议

- 全局层只写所有智能体都适用的规则
- 团队层写协作和交付约定，不重复成员身份
- Agent 层保留角色特有的人格和边界
- 把短小、常用内容放在 L0/L1，把长解释和示例放在 L2
- 不要依靠提示词替代工具权限、审批策略或代码级安全限制
