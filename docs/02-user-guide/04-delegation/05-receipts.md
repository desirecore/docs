# 执行回执

回执（Receipt）是 DesireCore 的核心机制之一——每次智能体执行操作后，都会自动生成一份完整的执行记录。它是你委派任务的"收据"。

## 回执是什么

回执就是一份详细的"做事报告"，记录了智能体在一次任务中做了什么、为什么这么做、用了什么工具、结果是什么。

你可以把它类比为：
- 银行的交易回单
- 快递的签收回执
- 代码提交的 commit 记录

**有了回执，委派才是可信的。** 没有回执的委派就像把钱交给别人但没拿到收据——你不知道钱花在了哪里。

## 回执包含哪些信息

一份完整的回执包含以下内容：

### 基本信息

```
┌──────────────────────────────────────────────────────┐
│  📋 执行回执                                           │
├──────────────────────────────────────────────────────┤
│                                                        │
│  回执编号：RCPT-20250315-143022                        │
│  任务：审查采购合同 XX-2025-001                         │
│  执行时间：2025-03-15 14:30 - 14:35                    │
│  总耗时：5 分 12 秒                                     │
│  状态：✅ 已完成                                        │
│                                                        │
└──────────────────────────────────────────────────────┘
```

### 输入与输出摘要

```
│  📥 输入                                                │
│  用户指令："帮我审查附件中的采购合同"                    │
│  附件：XX科技采购合同.pdf（128KB）                      │
│                                                        │
│  📤 输出                                                │
│  审查报告已保存至：docs/审查报告-XX-2025-001.md         │
│  发现 2 项风险，4 项合规                                │
│  综合风险等级：中等                                     │
```

### 步骤执行明细

```
│  📝 执行步骤                                            │
│                                                        │
│  1. ⚙️ [固化] 解析合同文件                  ✅ 0.8s    │
│  2. ⚙️ [固化] 检查违约金比例                ✅ 0.2s    │
│     └─ 15% ≤ 20% → 合规                               │
│  3. ⚙️ [固化] 检查付款周期                  ⚠️ 0.2s    │
│     └─ 15 天 < 30 天 → 风险                            │
│  4. 🧠 [灵活] 分析进口设备条款              ⚠️ 1.5min  │
│     └─ 缺少中文说明书条款                               │
│  5. 🧠 [灵活] 综合风险评估                  ✅ 1.2min  │
│  6. 🚪 [人闸门] 确认后保存报告              ✅ 3.5s    │
│     └─ 用户确认：已批准                                 │
```

### 依据追溯

```
│  📚 依据                                                │
│                                                        │
│  引用的规则：                                            │
│  - 违约金标准（≤20%）—— 来自 3 月 1 日教学              │
│  - 付款周期标准（≥30 天）—— 来自 3 月 1 日教学          │
│  - 进口设备中文说明书 —— 来自 3 月 5 日教学              │
│                                                        │
│  参考的示例：                                            │
│  - 审查报告模板 —— 来自 3 月 2 日教学                   │
```

### 变更记录

```
│  📂 文件变更                                            │
│                                                        │
│  新建文件：docs/审查报告-XX-2025-001.md                 │
│  变更大小：2.1KB                                        │
│  [查看文件内容]                                          │
```

## 如何查看回执

### 在对话中查看

任务完成后，智能体会在对话中展示回执摘要。你可以点击展开查看完整信息。

### 查看历史回执

```
你："显示最近 5 条回执。"

智能体："最近的执行回执：

        1. RCPT-20250315-143022  审查采购合同        ✅ 完成
        2. RCPT-20250315-140055  撰写会议纪要        ✅ 完成
        3. RCPT-20250314-163200  整理周报数据        ✅ 完成
        4. RCPT-20250314-100830  翻译技术文档        ✅ 完成
        5. RCPT-20250313-153045  发送客户邮件        ✅ 完成

        点击回执编号查看详情。"
```

### 搜索回执

```
你："找一下上周关于合同审查的所有回执。"
```

## 如何解读回执

重点关注以下几个方面：

| 关注点 | 为什么重要 |
|--------|-----------|
| **步骤类型分布** | 固化步骤多说明任务规则化程度高，灵活步骤多说明 AI 判断成分较大 |
| **人闸门触发次数** | 触发过多可能说明权限配置过严，触发过少可能存在安全隐患 |
| **警告和风险项** | 需要重点检查的内容 |
| **依据追溯** | 确认智能体是否基于正确的规则做出判断 |
| **执行耗时** | 异常的长耗时可能意味着任务复杂度超出预期 |

## 通过回执进行回滚

如果你对执行结果不满意，可以基于回执进行回滚。DesireCore 支持多个粒度的回滚：

| 回滚级别 | 含义 | 适用场景 |
|---------|------|---------|
| **单步回滚** | 撤销某一个步骤的结果 | 某个步骤的结果不对，其他步骤没问题 |
| **回合回滚** | 撤销最近一轮对话的所有操作 | 整个任务结果不满意 |
| **会话回滚** | 恢复到会话开始时的状态 | 整个会话的操作都需要重来 |
| **版本回滚** | 恢复到任意历史版本 | 需要回到更早的状态 |

```
你："回滚这次合同审查的第 4 步结果。"

智能体："将撤销步骤 4（分析进口设备条款）的结果。
        具体变更：
        - 审查报告中将移除进口设备相关的分析内容
        - 综合风险评估将重新计算

        确认回滚？"
```

:::tip 回滚前先查看回执
在决定回滚之前，先仔细查看回执中的步骤明细和依据追溯。有时候问题不是出在执行上，而是出在规则上——这时候修改规则比回滚更有效。
:::
