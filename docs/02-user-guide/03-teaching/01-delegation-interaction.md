# 什么是委派式交互

## 从"下指令"到"带徒弟"

想象一下你刚入职一家新公司，你的导师不会丢给你一本几百页的操作手册然后说"自己看着办"。好的导师会：

- 先告诉你**为什么**要这么做
- 然后**示范**给你看
- 让你试着做，做错了**纠正**你
- 最终你变成了能独立接活的人

DesireCore 的委派式交互（Delegation Interaction）就是这样一种理念：**你和智能体的关系不是"人操作工具"，而是"师傅带徒弟"。**

:::tip 核心区别
传统 AI：你每次都要从头说清楚你要什么，AI 给你一个结果，下次还是从头来。

DesireCore：你教一次，智能体就记住了。下次遇到类似情况，它会按你教的方式来做。你教得越多，它越可靠。
:::

## 与传统 AI 对话有什么不同

| 维度 | 传统 AI 对话 | DesireCore 委派式交互 |
|------|-------------|---------------------|
| **关系** | 你是用户，AI 是工具 | 你是师傅，智能体是徒弟 |
| **记忆** | 每次对话从零开始 | 教过的东西会被记住 |
| **行为** | 只会"说"，不会"做" | 能接任务、能执行、能交回执 |
| **成长** | 用100次和用1次一样 | 越用越懂你，越来越可靠 |
| **透明度** | 不知道 AI 怎么想的 | 每一步都可追溯、可审阅 |
| **可控性** | 结果不满意只能重新问 | 可以暂停、纠正、回滚 |

## 委派式交互的三大特点

### 1. 可教学

你可以用自然语言把"怎么做事"教给智能体。不需要写代码，不需要拖拽流程图。就像跟同事说话一样，把你的经验、标准和偏好告诉它。

> "审合同的时候，先看违约条款，如果违约金超过合同金额的30%，就标记为高风险。"

智能体会把这条规则记下来，下次审合同时自动应用。

### 2. 可委派

教会之后，你可以把任务交给智能体去做。它会先制定计划、征求你的确认，然后在你允许的权限范围内执行。

> "帮我审一下这份采购合同。"

智能体不会立刻开始——它会先告诉你它打算怎么做，等你确认后才动手。

### 3. 可审计

智能体做的每一件事都有据可查。它会生成详细的执行回执（Receipt），记录：做了什么、为什么这么做、用了什么工具、结果是什么。

如果结果不对，你可以追溯到具体的步骤，甚至回滚到之前的状态。

## 一个简单的类比

把 DesireCore 想象成一个"数字学徒培养系统"：

```
第一天：  你教它基本规则（"审合同要看这几点"）
第一周：  你给它示范（"看，这种条款就是有风险的"）
第一个月：它能独立处理简单任务，复杂的会来问你
第三个月：它已经是个可靠的助手，大部分工作可以放心交给它
```

而这一切的基础，就是接下来要介绍的**六原语**——它们定义了你和智能体之间交互的六种基本方式。
