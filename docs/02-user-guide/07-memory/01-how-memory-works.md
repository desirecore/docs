# 智能体如何记住你

你有没有这样的体验：每次找客服都要重复说一遍自己的需求，仿佛对方完全没有"记忆"？

DesireCore 的记忆系统（Memory System）就是为了解决这个问题。它让智能体不仅能完成当下的对话，还能**积累经验**，真正做到"越用越懂你"。

## 记忆 vs 聊天记录

你可能会想："不就是保存聊天记录吗？"实际上，记忆和聊天记录有本质区别：

| 对比维度 | 聊天记录 | 智能体记忆 |
|---------|---------|-----------|
| 本质 | 逐条消息的流水账 | 提炼后的经验和认知 |
| 信息密度 | 低——包含大量无关内容 | 高——只保留有价值的信息 |
| 影响行为 | 不影响 | 直接影响智能体的回应方式 |
| 可管理性 | 只能整段查看 | 可以逐条编辑、删除、标记 |

打个比方：聊天记录像是监控录像，记忆则是你的**个人笔记本**——智能体会从每次对话中提炼出要点，记录下来，下次交流时翻阅参考。

## 记忆从哪里来

智能体的记忆不需要你手动输入。在每次对话结束时，系统会自动进行一个叫做**复盘（Reflect）**的过程：

1. **对话结束** — 你和智能体完成一次交流
2. **自动复盘** — 智能体分析本次对话，提取有价值的信息
3. **生成候选记忆** — 系统生成若干条候选记忆条目
4. **你来审阅** — 你可以接受、拒绝或编辑这些条目
5. **正式记忆** — 通过审阅的条目写入智能体的记忆库

:::tip 你始终掌控
智能体不会偷偷记住什么。每一条新记忆都会经过你的审阅，你可以随时查看、修改或删除任何记忆。
:::

## 记忆如何影响智能体

一旦某条信息被记忆，它会在未来的对话中发挥作用：

- **个性化回应**：智能体知道你喜欢简洁还是详细的回答风格
- **上下文延续**：不需要每次重复解释项目背景
- **主动关怀**：智能体可能在合适时机提醒你之前提到的截止日期
- **决策参考**：之前讨论过的方案和决策可以被直接引用

例如，如果你告诉法律顾问智能体"我是一家科技公司的 CTO"，它不仅会记住这个事实，还会在后续的法律咨询中自动从技术管理者的角度提供更贴合的建议。

## 记忆的"大脑"结构

智能体的记忆并不是一团混乱的信息，而是有条理地组织在三个不同的区域——我们称之为**三域记忆模型**。你可以把它想象成人脑的不同功能分区：

- **核心记忆** — 好比智能体的"性格"，是它出厂时就带有的专业知识和基本认知
- **关系记忆** — 好比智能体和你之间的"日记"，记录了你们互动中积累的经验
- **共享记忆** — 好比团队的"共享文档"，多个智能体都可以参考的公共知识

## 记忆的生命周期

记忆不是一成不变的。随着时间推移，智能体的记忆会经历自然的演变：

- **活跃的记忆** 会被优先检索
- **久未使用的记忆** 会逐渐被压缩归档
- **不再相关的记忆** 最终可能被清理

这个过程模仿了人类记忆的自然遗忘机制——确保智能体的"脑容量"不会被过时的信息占满，同时重要的记忆会被妥善保留。

:::tip 永不遗忘
如果某条记忆对你特别重要，你可以将它标记为"固定"（Pin），这样它就永远不会被遗忘。
:::
