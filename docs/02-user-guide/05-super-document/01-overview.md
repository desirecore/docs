# 超级文书是什么

你用过 Code Review 吗？开发者提交代码后，同事逐行审阅，接受或拒绝每一处修改。**超级文书（Super Document）** 把这套经过数十年验证的协作机制搬到了文档写作中——AI 帮你写，你来审。

## 核心理念：像 Code Review 一样审文档

传统 AI 写作工具的问题在于：AI 给你一大段文字，你不知道它改了什么、为什么改。接受还是重写？你只能凭感觉。

超级文书的做法完全不同：

| 传统 AI 写作 | 超级文书 |
|---|---|
| AI 输出一整篇文档，你自己找差异 | AI 的每一处修改都标记出来 |
| 只能整体接受或重写 | 可以逐条接受、拒绝或编辑 |
| 不知道 AI 为什么这么改 | 每次修改附带理由说明 |
| 改错了只能从头再来 | 完整版本历史，随时回滚 |

## 超级文书与普通文本编辑的区别

超级文书不是又一个 Markdown 编辑器。它的核心价值在于**修改过程的可视化和可控性**：

- **可见**：AI 做了什么修改，一目了然
- **可控**：你决定接受哪些、拒绝哪些
- **可逆**：任何操作都可以撤销，任何版本都可以恢复
- **可解释**：AI 不只是改，还告诉你为什么改

:::tip 一句话理解
如果说对话是你和智能体的"口头交流"，那超级文书就是你们的"书面协作"——所有修改留痕，所有决策可追溯。
:::

## 适用场景

超级文书适合任何需要 AI 参与的文档工作：

- **日常写作**：邮件、报告、方案，让 AI 帮你润色和改进
- **专业文书**：合同、标书、法律文件，AI 辅助撰写，你把关审核
- **团队协作**：多人参与的文档，通过版本历史追踪每次变更
- **知识沉淀**：将零散想法整理成结构化文档

## 一图看懂超级文书流程

```
你发起写作任务
    |
    v
智能体生成或修改内容
    |
    v
Diff 视图展示所有变更  <-- 红色 = 删除，绿色 = 新增
    |
    v
你逐条审阅
    |
    +--> 接受：修改生效
    +--> 拒绝：恢复原文
    +--> 编辑：手动调整
    |
    v
确认后形成新版本
    |
    v
继续对话，进一步迭代...
```

每一轮修改都会被记录为一个版本。你可以随时回到任何历史版本，就像代码的 Git 历史一样。
