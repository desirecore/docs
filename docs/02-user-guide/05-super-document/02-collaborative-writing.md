# 协作写文档

超级文书的核心体验是**与智能体协作**——你提需求，智能体写初稿；你审改，智能体再修改。这个过程可以反复迭代，直到文档达到你的要求。

## 如何开始一个文档写作任务

有两种方式进入超级文书：

### 方式一：从对话中发起

在与智能体的对话中，直接描述你的写作需求。当智能体识别到你需要生成或修改文档时，会自动切换到超级文书模式。

例如：

> "帮我写一份项目周报，包含本周完成的三个功能点和下周计划。"

> "把这份合同的第三条改得更严谨一些。"

### 方式二：打开已有文档

如果你已经有一份文档，可以在超级文书编辑器中打开它，然后通过右侧的对话区域向智能体发出修改指令。

## 与智能体的协作流程

一次典型的协作流程如下：

### 第一步：描述需求

在对话区域告诉智能体你想要什么。越具体，结果越好：

:::tip 好的需求描述
- "帮我把第二段改成正式的商务语气，但保留原有的数据和论点。"
- "在结尾增加一个总结段落，提炼出三个核心观点。"
- "检查全文，修正所有语法错误和不通顺的地方。"
:::

:::warning 模糊的需求描述
- "帮我改改。"（改什么？怎么改？）
- "写好一点。"（好的标准是什么？）
:::

### 第二步：查看修改

智能体完成修改后，所有变更会以 Diff 视图呈现在文档区域。你可以看到：

- **红色标记**：被删除的内容
- **绿色标记**：新增的内容
- **修改理由**：每处修改旁边会附带说明

### 第三步：审阅决策

对于每一处修改，你可以：

- **接受** — 认可这个修改，让它生效
- **拒绝** — 不同意，恢复原文
- **编辑** — 在 AI 修改的基础上进一步调整

### 第四步：继续迭代

审阅完成后，你可以继续对话，提出新的修改要求。智能体会基于当前最新的文档内容进行下一轮修改。

## 文档格式支持

超级文书基于 Markdown 格式，支持以下内容类型：

| 内容类型 | 支持情况 | 说明 |
|---|---|---|
| 标题和段落 | 完整支持 | 多级标题、段落、引用 |
| 列表 | 完整支持 | 有序列表、无序列表、嵌套列表 |
| 表格 | 完整支持 | 标准 Markdown 表格 |
| 代码块 | 完整支持 | 语法高亮、多语言 |
| 图片 | 完整支持 | 本地图片和在线图片 |
| 流程图 | 完整支持 | Mermaid 语法 |
| 数学公式 | 完整支持 | LaTeX 语法 |

:::info 关于导出
完成文档后，你可以导出为 docx 或 PDF 格式，方便在其他场景使用。详见 [版本历史](./05-version-history.md)。
:::

## 编辑和修改

除了通过对话让智能体修改，你也可以直接在编辑器中手动编辑文档内容。手动编辑的内容同样会被记录在版本历史中。

**智能体修改和手动编辑可以混合使用**——比如让智能体生成初稿后，你手动微调措辞，再让智能体补充一个段落。
