# 逐条审阅

审阅是超级文书的核心交互。智能体提出修改建议后，你可以逐条审查每一处变更，决定接受、拒绝或进一步编辑。

## 审阅模式介绍

当智能体完成修改后，超级文书自动进入审阅模式。此时：

- 所有修改以 Diff 形式高亮显示
- 第一处修改被自动选中
- 编辑器顶部显示审阅进度条（例如"0/8 已审阅"）
- 每处修改旁边出现操作按钮

你可以从任意一处修改开始审阅，不必按顺序。

## 接受/拒绝单条修改

对于每一处修改，你有三个选项：

### 接受

点击绿色的"接受"按钮（或按 `Enter`），该修改立即生效：

- 如果是新增内容，绿色高亮消失，内容融入正文
- 如果是删除内容，红色标记的内容被移除
- 如果是替换，原文被新内容替代

### 拒绝

点击红色的"拒绝"按钮（或按 `Backspace`），恢复原文：

- 新增的内容被移除
- 删除的内容被恢复
- 替换的内容恢复为原文

### 编辑

点击"编辑"按钮（或按 `E`），在智能体修改的基础上手动调整：

1. 该处修改进入可编辑状态
2. 你可以修改文字内容
3. 按 `Esc` 取消编辑，按 `Enter` 确认

:::tip 什么时候用编辑？
当智能体的修改方向是对的，但措辞还需要微调时——比如 AI 把"好"改成了"卓越"，你觉得"优秀"更合适。
:::

## 批量操作

如果修改数量较多，你可以使用批量操作提高效率：

| 操作 | 快捷键 | 说明 |
|---|---|---|
| 接受全部 | `Ctrl+Shift+Enter` | 一键接受所有待审阅的修改 |
| 拒绝全部 | `Ctrl+Shift+Backspace` | 一键拒绝所有待审阅的修改 |

:::warning 谨慎使用批量操作
批量接受或拒绝操作是可以撤销的（`Ctrl+Z`），但建议至少快速浏览一遍再做决定。
:::

你也可以先逐条审阅重要的修改，然后对剩余的修改使用批量操作。

## 添加评论和反馈

审阅过程中，你可以针对某一处修改添加评论。这些评论会被传递给智能体，作为下一轮修改的参考：

1. 选中一处修改
2. 点击"评论"按钮
3. 输入你的反馈，例如："这里改得太正式了，保持原来的语气就好。"
4. 智能体会在下一轮修改中参考你的评论

:::info 评论不只是给智能体看
评论也会保留在版本历史中。如果你在团队中使用超级文书，其他人可以看到你的审阅意见。
:::

## 快捷键速查

| 操作 | 快捷键 |
|---|---|
| 接受当前修改 | `Enter` |
| 拒绝当前修改 | `Backspace` |
| 编辑当前修改 | `E` |
| 接受全部 | `Ctrl+Shift+Enter` |
| 拒绝全部 | `Ctrl+Shift+Backspace` |
| 跳转到下一个修改 | `]` |
| 跳转到上一个修改 | `[` |
