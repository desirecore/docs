# Diff 可视化

超级文书的 Diff 视图让你清楚看到智能体修改了什么。每一处新增、删除和替换都用颜色标记出来，就像代码的 diff 一样直观。

## 什么是 Diff 视图

当智能体修改你的文档后，超级文书不会直接替换原文，而是以 Diff（差异）的形式展示修改前后的对比。这样你可以：

- 精确看到哪些内容被改动了
- 理解修改的范围和影响
- 逐条决定是否接受

## 颜色编码含义

Diff 视图使用四种颜色区分不同类型的变更：

| 变更类型 | 颜色 | 标记方式 | 含义 |
|---|---|---|---|
| **删除** | 红色 | 红色背景 + 左侧红色竖线 | 这部分内容被移除 |
| **新增** | 绿色 | 绿色背景 + 左侧绿色竖线 | 这是新加入的内容 |
| **替换** | 红 + 绿组合 | 先红后绿，渐变过渡 | 原内容被替换为新内容 |
| **移动** | 蓝色 | 蓝色背景 + 箭头指示 | 内容位置发生了变化 |

:::tip 快速阅读技巧
- 红色部分是"旧的"，绿色部分是"新的"
- 如果一段同时有红和绿，说明这是一处替换——红色是原文，绿色是修改后的内容
- 蓝色箭头表示内容被移动到了其他位置
:::

## 三种视图模式

根据修改的类型和你的阅读习惯，超级文书提供三种视图模式：

### 内联模式（Inline）

删除和新增内容在同一行展示。适合小范围的措辞调整：

```
这个方案的实施周期为[-三个月-]{+六个月+}，
需要[-5 人-]{+8 人+}的团队配置。
```

> 移动端默认使用此模式。

### 分栏模式（Side by Side）

左侧显示原文，右侧显示修改后的内容。适合大范围重写，方便逐段对比：

```
原文（左侧）              |  修改后（右侧）
--------------------------|---------------------------
项目于去年启动，           |  项目于 2025 年 Q3 启动，
目前进展顺利。             |  目前已完成 80% 的里程碑。
```

> PC 端默认使用此模式。

### 统一模式（Unified）

仅显示最终结果，变更部分高亮标注。适合快速浏览整体效果：

> 此模式也用于导出前的预览。

你可以在编辑器顶部的工具栏中随时切换视图模式。

## 在 Diff 中导航

当智能体做了多处修改时，你可以使用快捷键快速在变更之间跳转：

| 操作 | 快捷键 | 说明 |
|---|---|---|
| 跳转到下一个修改 | `]` | 光标移动到下一处变更 |
| 跳转到上一个修改 | `[` | 光标移动到上一处变更 |

:::info 修改计数
编辑器顶部会显示当前待审阅的修改总数（例如"3/12 处修改"），帮助你了解审阅进度。
:::

## 修改理由

每处修改旁边都会附带一个理由说明，告诉你智能体为什么做这个修改。一条合格的修改理由包含三个要素：

| 要素 | 含义 | 示例 |
|---|---|---|
| **改动目的** | 为什么修改 | "使表达更正式、符合商务文书规范" |
| **依据来源** | 基于什么做的决策 | "参考用户偏好：正式语气" |
| **影响范围** | 修改了哪里 | "仅影响第二段，不改变整体结构" |

你可以点击修改旁边的理由图标查看完整说明。如果理由不充分，这可以成为你拒绝该修改的依据。
