# 版本历史

超级文书的每一次修改都会被记录为一个版本。你可以随时查看完整的修改历史、对比任意两个版本、或回滚到之前的状态。

## 查看文档的版本历史

点击编辑器右上角的"版本历史"按钮，打开版本时间线面板。你会看到：

- **版本列表**：按时间倒序排列，最新的版本在最上方
- **版本摘要**：每个版本附带一句简短说明（例如"润色第二段措辞"）
- **操作人**：标记是智能体修改还是你手动编辑
- **时间戳**：精确到分钟

超级文书的版本分为三个粒度层级：

| 粒度 | 说明 | 示例 |
|---|---|---|
| **Patch（补丁）** | 单次修改操作 | "将'好'替换为'优秀'" |
| **Turn（轮次）** | 一轮对话中的所有修改 | "根据用户要求，润色第 2-4 段" |
| **Session（会话）** | 一次完整写作会话的所有变更 | "2026-03-05 下午的写作会话" |

你可以在这三个层级之间切换查看，按需选择合适的粒度。

## 版本比较

选中任意两个版本，点击"对比"按钮，超级文书会以 Diff 视图展示两个版本之间的差异：

1. 在版本列表中点击第一个版本（标记为"基准"）
2. 点击第二个版本（标记为"比较"）
3. 编辑器区域切换为分栏对比视图

这在以下场景特别有用：

- 查看一轮修改的完整影响
- 比较智能体连续两次修改的差异
- 评估一次会话前后文档的变化

## 回溯到历史版本

如果当前的文档不满意，你可以回溯到任何历史版本：

1. 在版本列表中找到目标版本
2. 点击"预览"查看该版本的完整内容
3. 确认后点击"恢复到此版本"

:::warning 回溯不会丢失历史
回溯操作本身也会被记录为一个新版本。这意味着你随时可以再恢复到回溯之前的状态——没有任何操作是不可逆的。
:::

三个粒度的回滚对应不同场景：

| 回滚粒度 | 适用场景 | 操作 |
|---|---|---|
| 撤销单个 Patch | 某一处修改不对，其余保留 | 在版本历史中选中该 Patch，点击"撤销" |
| 回滚到某个 Turn | 一轮修改整体不行，回到上一轮 | 选中目标 Turn 的起点版本，点击"恢复" |
| 回滚到某个 Session | 这次写作会话偏离方向 | 选中 Session 起始版本，点击"恢复" |

## 版本管理最佳实践

:::tip 养成好习惯
1. **分步修改，分步审阅**：不要一次让智能体改太多。分段修改更容易追踪和控制。
2. **重要节点手动标记**：在满意的版本上添加备注，方便日后查找。
3. **先审后改**：完成一轮审阅后再发起下一轮修改，避免版本混乱。
4. **导出前检查**：在导出最终文档前，切换到"统一模式"通读全文。
:::

## 导出文档

当文档完成后，你可以导出为以下格式：

| 格式 | 说明 |
|---|---|
| **docx** | Word 文档格式，支持标题、段落、列表、图片和表格 |
| **PDF** | 通过 Markdown 渲染生成 PDF |

点击编辑器右上角的"导出"按钮，选择目标格式即可。
