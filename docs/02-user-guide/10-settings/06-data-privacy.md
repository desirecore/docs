# 数据与隐私

DesireCore 采用**本地优先**（Local-first）架构，你的数据默认存储在本地设备上。本页介绍如何管理你的数据。

## 数据存储位置

DesireCore 的数据默认存放在以下目录：

| 操作系统 | 路径 |
|---------|------|
| macOS | `~/.desirecore/` |
| Windows | `%USERPROFILE%\.desirecore\` |
| Linux | `~/.desirecore/` |

该目录包含：

- **智能体配置**：`agents/` — 智能体的人设、记忆和设置
- **技能定义**：`skills/` — 全局和智能体专属技能
- **用户配置**：`config/` — 用户档案、头像、算力配置
- **对话记录**：会话历史和执行回执
- **市场缓存**：`market/` — 从市场下载的资源

## 导出数据

你可以将 DesireCore 的数据导出为备份文件，用于迁移到另一台设备或定期备份。

1. 打开 **设置** → **数据与隐私**
2. 点击 **导出数据**
3. 选择保存位置
4. 等待导出完成

导出文件包含你的智能体配置、对话记录、技能和用户设置。

:::tip
建议定期导出数据作为备份，特别是在升级系统或更换设备之前。
:::

## 导入数据

从备份文件恢复数据：

1. 打开 **设置** → **数据与隐私**
2. 点击 **导入数据**
3. 选择之前导出的备份文件
4. 等待导入完成，应用会自动重启

:::warning
导入操作会覆盖当前的本地数据。如果当前设备上已有重要数据，建议先导出一份备份。
:::

## 清除本地数据

如果你需要重置 DesireCore 到初始状态，可以选择性地清除本地数据。

1. 打开 **设置** → **数据与隐私**
2. 点击 **清除本地数据**
3. 选择要清除的数据类别：

| 数据类别 | 包含内容 |
|---------|---------|
| **对话记录** | 所有聊天消息、会话历史和执行回执 |
| **智能体** | 智能体配置、人设和记忆（核心智能体会自动重装） |
| **技能** | 全局技能定义（内置技能会自动重装） |
| **算力** | LLM 提供商、模型列表和计算节点配置 |
| **存储** | S3 对象存储连接配置 |
| **代码托管** | GitHub/Gitee/Gitea 连接配置 |
| **用户档案** | 用户身份、头像、个人记忆库和命令审批设置 |
| **市场数据** | 官方市场仓库缓存 |
| **缓存与日志** | 用量统计索引、关系图谱缓存和日志文件 |

4. 确认选择后，点击 **清除选中数据**
5. 系统会依次停止后台服务、删除数据、重启应用

:::danger
数据清除操作**不可恢复**。被清除的数据将被永久删除。如果你只是想解决应用异常，建议先尝试清除"缓存与日志"类别，这通常能解决大部分问题且不会丢失重要数据。
:::

## 隐私保障

DesireCore 在隐私保护方面的设计原则：

- **数据不出本地**：所有数据默认存储在你的设备上，不会自动上传到云端
- **API Key 安全存储**：使用操作系统的凭据管理器加密存储
- **AI 调用透明**：每次调用 AI 模型的请求和响应都有审计记录，你可以在活动记录中查看
- **无遥测收集**：DesireCore 不收集使用遥测数据
