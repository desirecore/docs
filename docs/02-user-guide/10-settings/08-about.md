# 关于与更新

在"关于"页面你可以查看当前版本、检查更新，以及查阅使用条款和隐私政策。

## 查看当前版本

1. 打开 **设置** 面板
2. 滚动到最底部的 **关于** 分区
3. **版本** 一行显示当前安装的版本号

## 检查更新

:::info
检查更新功能在桌面客户端中可用。
:::

### 自动检查

DesireCore 会在启动后**每 5 分钟**自动向更新服务器查询是否有新版本。你无需手动操作，系统会在后台静默完成检查。

### 手动检查

1. 在关于分区中，点击 **版本** 一行
2. 右侧会显示"检查更新"，点击后开始检查
3. 检查结果：
   - **已是最新版本**：你正在使用最新版
   - **新版本 vX.X.X**：有可用更新，系统会自动开始下载
   - **检查失败，点击重试**：网络异常，稍后再试

### 更新流程

当检测到新版本后：

1. 系统自动从最快的下载源下载更新包，右侧显示下载进度百分比
2. 下载完成后，弹出提示 **"新版本已下载完成，是否现在重启？"**
3. 选择 **立即重启** 即可安装更新；选择 **稍后** 则在 24 小时内不再弹出提示

:::tip 多源智能下载
DesireCore 维护了多个下载源（CDN、构建服务器、GitHub Releases 等）。下载前会自动对各源进行**测速**，选择最快的源下载安装包，提升下载速度和可靠性。如果某个源不可用，会自动切换到其他源。
:::

### 灰度发布

新版本通常会通过**灰度发布**逐步推送给用户。这意味着新版本不会一次性推送给所有人，而是先推送给一部分用户，确认稳定后再扩大推送范围。如果你暂时没有收到某个版本的更新，属于正常现象，请耐心等待。

## 更新通道

DesireCore 提供两个更新通道：

| 通道 | 说明 |
|------|------|
| **Stable（稳定版）** | 默认通道。经过充分测试的正式版本，适合大多数用户 |
| **Beta（测试版）** | 抢先体验最新功能，可能包含未完全验证的变更。适合愿意帮助测试的用户 |

### 切换更新通道

1. 打开 **设置** 面板
2. 在 **关于** 分区中找到 **更新通道**
3. 选择 **Stable** 或 **Beta**

切换通道后，下次自动检查时将使用新通道获取更新。

:::caution
切换到 Beta 通道可能会收到尚未完全稳定的版本。如遇到问题，可随时切回 Stable 通道。切回后不会自动降级，而是会在 Stable 通道发布更新版本时收到推送。
:::

## 法律文档

在关于分区中，你还可以查阅：

- **使用条款**：DesireCore 的使用协议
- **隐私政策**：数据收集和处理说明

点击对应项即可查看完整文档。
