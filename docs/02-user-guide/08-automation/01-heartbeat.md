# 心跳监控

传统的 AI 助手只会被动等待你的指令。DesireCore 的智能体则不同——它们可以**主动巡检**外部环境，发现变化时**克制地通知你**。这个能力的核心就是心跳系统（Heartbeat System）。

## 什么是心跳

心跳，就是智能体定期"睁开眼睛看一看"的过程。就像值班保安每隔一段时间巡视一圈一样，智能体会按照设定的频率检查它关注的数据源，然后决定要不要告诉你。

每次心跳的流程：

1. **检查数据源** — 扫描邮箱、GitHub、日历等已连接的服务
2. **判断变化** — 对比上次检查的状态，识别有没有新内容
3. **决定响应** — 根据变化的重要程度，选择静默、通知还是请求操作

## 三级响应

心跳检查的结果分为三个级别，每个级别有不同的行为：

### OK — 一切正常

当所有数据源都没有变化时，智能体会静默记录一条检查回执，**不打扰你**。

你可以在心跳历史中看到这些"OK"记录，知道智能体一直在默默工作。

### Notify — 有新消息

当检测到变化但不需要你立即采取行动时，智能体会发送一条**信息通知**。

比如："收到 3 封新邮件，GitHub 上 PR #42 有新评论，下午 2 点有项目周会。"

你可以：
- 立即查看详情
- 点击"稍后提醒"（Snooze），智能体会在 15 分钟、1 小时或 3 小时后再次提醒

### Action — 需要你的确认

当检测到需要你亲自决策或执行外部操作的变化时，智能体会发送**操作请求**。

比如："CEO 发来邮件，需要你在下班前回复预算确认。"

操作请求会显示：
- 需要执行的操作描述
- 操作的风险等级（低/中/高）
- 你的选择：**确认执行** / **拒绝** / **稍后处理**

:::warning 安全第一
需要执行外部操作的请求**永远不会跳过你的确认**。智能体不会擅自发送邮件、审批 PR 或执行任何有实际影响的操作。
:::

## 配置巡检

### 巡检频率

你可以调整智能体检查数据源的频率：

- **最短间隔**：由智能体开发者设定（通常为 5 分钟）
- **最长间隔**：最多 24 小时
- 你可以在这个范围内自由设定

频率的选择取决于你的需求——需要及时响应的场景可以设短一些，不那么紧急的场景可以设长一些。

### 数据源开关

每个智能体可能连接了多个数据源。你可以针对每个数据源单独开启或关闭巡检：

- **邮箱检查** — 开启/关闭
- **GitHub 检查** — 开启/关闭
- **日历检查** — 开启/关闭

:::info 能添加新数据源吗
目前，你只能启用或禁用智能体预定义的数据源。添加新的数据源需要通过安装相应的工具或 MCP 服务来实现。
:::

## 静音策略

智能体虽然会主动通知，但它也懂得"克制"。你可以设置多种静音策略：

| 策略 | 说明 | 默认值 |
|------|------|--------|
| **安静时段（Quiet Hours）** | 在此时间段内只记录不通知 | 23:00 - 08:00 |
| **通知冷却（Cooldown）** | 同级别通知之间的最短间隔 | 30 分钟 |
| **专注模式（Focus Mode）** | 全面静音，仅 Action 级仍可推送 | 关闭 |
| **临时静音（Snooze）** | 暂时屏蔽通知，到点恢复 | 关闭 |

**安静时段**期间，智能体照常执行心跳检查，但不会推送任何 Notify 级通知。所有变化会被记录下来，等安静时段结束后的第一次心跳时合并汇报。

**专注模式**下，只有 Action 级的紧急操作请求才会打扰你，其他通知全部被静默。

## 查看巡检结果

### 心跳状态卡片

在智能体详情页，你可以看到实时的心跳状态：

- 上次检查时间
- 下次预计检查时间
- 今日检查次数
- 状态分布（OK / Notify / Action 各多少次）

### 心跳历史

历史记录展示最近的所有心跳检查，支持按级别筛选。每条记录包括：

- 检查时间
- 响应级别
- 变化摘要
- 处理状态（如 Action 级是否已确认）

:::tip 可追溯
每次心跳检查都会生成一条不可篡改的回执（Receipt），确保智能体的所有行为可追溯、可审计。
:::
