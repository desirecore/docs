# 通知管理

智能体需要在合适的时机把重要的信息传达给你。DesireCore 设计了一套分层通知体系，确保重要的消息不被错过，同时避免不必要的打扰。

## 三级通知体系

DesireCore 的通知分为三个级别，视觉表现和行为各不相同：

### 普通通知

日常信息类通知，不需要立即处理。

- **来源**：心跳检查的 Notify 级结果、调度任务完成报告等
- **视觉**：标准消息卡片
- **声音**：无
- **行为**：出现在对话流中，你可以随时查看

### 重要通知

需要你关注但不紧急的通知。

- **来源**：重要邮件到达、PR 审核请求、日程变更等
- **视觉**：突出显示的消息卡片
- **声音**：轻微提示音
- **行为**：出现在对话流中，并在通知中心显示角标

### 紧急通知

需要你立即决策或行动的通知。

- **来源**：心跳检查的 Action 级结果、到期提醒等
- **视觉**：醒目的操作请求卡片，附带操作按钮
- **声音**：明确的提醒音
- **行为**：弹出桌面通知，在应用中高亮显示

:::info 级别由智能体判断
通知的级别不是固定的——智能体会根据内容的性质和紧迫程度自主判断该用哪个级别通知你。同样是一封邮件，来自同事的日常沟通可能只是普通通知，来自 CEO 的紧急邮件则会被判定为紧急通知。
:::

## 通知设置

你可以在设置中精细化控制通知行为：

### 全局设置

| 设置项 | 说明 | 选项 |
|--------|------|------|
| 桌面通知 | 系统原生通知弹窗 | 开启 / 关闭 |
| 声音提示 | 通知伴随声音 | 开启 / 关闭 |
| 应用角标 | 应用图标上的未读数字 | 开启 / 关闭 |

### 按智能体设置

每个智能体的通知可以单独配置：

- 是否接收该智能体的通知
- 静音特定智能体（保留记录但不推送）
- 设置该智能体的安静时段

### 免打扰模式

当你需要专注工作时：

1. 开启**免打扰模式**
2. 所有普通和重要通知被静默
3. 仅紧急通知（Action 级）可以突破
4. 被抑制的通知在你关闭免打扰后统一呈现

:::tip 快速开启
你可以在应用顶部的状态栏快速切换免打扰模式，无需进入设置页面。
:::

## 通知历史

所有通知都会被记录在通知中心，方便你事后查看：

- **时间线视图** — 按时间顺序查看所有通知
- **按智能体筛选** — 只看某个智能体的通知
- **按级别筛选** — 只看重要或紧急的通知
- **搜索** — 在通知历史中搜索特定内容

通知中心还会显示哪些通知被静音策略抑制了（标记为"已抑制"），你仍然可以查看它们的内容。

## 操作请求的处理

对于 Action 级别的操作请求，你有三种处理方式：

| 操作 | 说明 |
|------|------|
| **确认执行** | 智能体将执行对应的操作 |
| **拒绝** | 操作不会执行，智能体记录你的决定 |
| **稍后处理** | 标记为待处理，下次心跳时再次提醒 |

每个操作请求都显示了操作的**风险等级**，帮助你做出判断：

- **低风险**（绿色）：如"标记邮件已读"
- **中风险**（橙色）：如"发送邮件回复"
- **高风险**（红色）：如"支付确认"、"删除文件"
