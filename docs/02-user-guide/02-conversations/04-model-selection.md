# 选择 AI 模型

DesireCore 支持接入多家 AI 模型供应商（Provider），你可以在对话中随时切换使用的模型。不同的模型在能力、速度和成本上各有特点，选择合适的模型可以让你获得更好的体验。

## 模型选择器

模型选择器位于聊天头部区域的操作按钮组中，以一个图层叠加图标显示。

**图标颜色含义：**

| 颜色 | 含义 |
|------|------|
| 灰色 | 使用默认模型，无手动覆盖 |
| 绿色 | 已手动选择模型，且该模型可用 |
| 橙色 | 已手动选择模型，但该模型当前不可用（供应商被禁用/移除等） |

点击图标打开模型选择下拉面板。

## 选择模型

下拉面板中的模型按供应商（Provider）和模型厂商分组展示：

1. **使用默认** --- 顶部选项，恢复为系统默认模型（不手动指定）
2. **各供应商分组** --- 每个供应商名称下列出该供应商可用的模型

每个模型条目显示：

- 模型显示名
- 服务类型标签（如 chat、completion）
- 能力标签（如 vision、tool use、reasoning、long context）
- 价格、credit 消耗倍率或相对成本（如有）
- 默认模型标识和验证状态

点击任一模型即切换当前对话使用的模型。选中状态用绿色圆点标识。

将鼠标悬停在模型上，可以查看模型信息卡。信息卡通常包含模型描述、上下文窗口、输入/输出/cache 价格、能力标签和来源说明。

:::tip 作用范围
模型选择是**当前会话级别**的临时覆盖，不会影响其他对话或全局默认设置。关闭对话后会恢复默认。
:::

## 思考深度

当所选模型声明支持 reasoning / thinking 能力时，输入区会显示 **思考深度**。你可以选择：

| 选项 | 行为 | 适合场景 |
|------|------|----------|
| **自动** | 不发送显式思考参数，跟随模型或供应商默认行为 | 不确定模型最佳设置时 |
| **关闭思考** | 尝试发送供应商支持的关闭参数 | 简单问答、延迟敏感任务 |
| **最浅 / 浅 / 中 / 深 / 最深** | 显式开启思考并指定相对深度 | 从日常分析到复杂推理、代码和长任务 |

默认档位为“中”。有效设置按 **当前会话选择 → 智能体模型设置 → 全局默认** 的顺序解析；不支持可控思考的模型会回退为“自动”，相应控件也会停用。

更深的档位通常会使用更多推理预算，响应更慢，并可能增加按量费用或订阅额度消耗。不同供应商对深度的支持并不完全一致：只有开/关能力的模型会把多个深度档位折算为“开启”，不支持“最深”的模型可能降级到“深”，“关闭”也只是供应商能力范围内的尽力关闭。切换后应以实际输出、延迟和供应商账单为准。

## 能力与可用性

模型选择器会根据模型能力和当前场景过滤模型：

- 已验证失败的模型会被隐藏或标记为不可用
- 已废弃或被供应商移除的模型不会作为推荐项出现
- 图像生成、视频生成、语音合成等非聊天模型不会作为对话模型选择
- 支持视觉输入的模型会显示对应能力标签，适合图片理解和扫描 PDF 任务
- 官方云端算力模型会显示 credit 或相对消耗信息

## 模型不可用警告

如果你之前选择的模型因为以下原因不可用，下拉面板顶部会显示橙色警告条：

- **供应商被禁用** --- 该模型的 Provider 在设置中被关闭了
- **供应商被移除** --- 该 Provider 已从配置中删除
- **模型被移除** --- 该模型已从 Provider 的可用列表中移除

出现警告时，建议你重新选择一个可用的模型，或点击「使用默认」恢复默认设置。

## API Key 未配置

如果某个 Provider 未配置 API Key，下拉面板中会显示橙色提示。该 Provider 的模型仍会列出，但可能无法正常使用。前往「设置 > 算力服务」配置对应的 API Key。

登录后，官方云端模型会自动以 `desirecore-cloud` 形式绑定到资源面板；这类模型不需要你手动填写 API Key，但会消耗账号中的 credit 或订阅额度。

## 配置默认模型

要修改全局默认使用的模型：

1. 打开「设置」（左下角齿轮图标）
2. 进入「算力服务」或通过资源管理器的「算力」入口
3. 在「AI 服务 > 默认映射」中设置各服务类型的默认模型

:::info 本地优先
DesireCore 的模型配置与 API Key 保存在本机；Key 的实际值位于 `~/.desirecore/config/secrets.json`。选择对应供应商执行请求时，Key 会作为鉴权信息发送给该供应商。请保护系统账户、磁盘和备份，详细边界见[算力服务配置](../10-settings/04-compute-service.md#管理-api-key)。
:::

## 常见场景的模型选择建议

| 场景 | 建议选择 | 理由 |
|------|---------|------|
| 日常对话 | 中等规模模型 | 响应快，成本低 |
| 复杂推理 / 长文分析 | 旗舰模型（如 Claude Opus、GPT-4o） | 推理能力强 |
| 代码辅助 | 代码优化模型（如 Claude Sonnet、DeepSeek Coder） | 编程能力突出 |
| 快速问答 | 轻量模型（如 Haiku、GPT-4o-mini） | 速度快，适合简单任务 |
