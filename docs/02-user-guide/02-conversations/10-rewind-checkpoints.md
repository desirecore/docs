# Rewind 与 Checkpoint

Rewind / Checkpoint 让你在智能体走错方向时回到之前的稳定状态。它不只是删除几条聊天消息，还可以恢复当时的文件、媒体、AgentFS、记忆和技能状态。

## 什么时候用

- 智能体误解需求，后续执行都偏了
- 一次大改动影响了文件或文档状态
- 想比较两个方案，但保留旧分支
- 需要撤回刚才的自动学习或技能变更

## 入口

你可以通过以下方式打开回撤：

- 在输入框输入 `/rewind`
- 在消息操作栏选择 Rewind
- 在聊天头部「更多」菜单中选择回撤
- 使用快捷键 `Cmd + Alt + Z`（macOS）或 `Ctrl + Alt + Z`（Windows/Linux）

## 回撤模式

| 模式 | 作用 |
|------|------|
| 仅对话 | 回到某条用户消息前的对话状态 |
| 仅状态 | 恢复文件、媒体、AgentFS、记忆或技能状态 |
| 对话 + 状态 | 同时恢复对话和工作状态 |

执行前会显示影响范围预览。确认后，DesireCore 会从目标检查点创建新的运行分支，旧分支仍保留在历史中。

## 自动检查点

DesireCore 会在关键操作前后自动建立检查点，例如：

- 用户发送新任务
- 智能体开始修改文件
- 写入记忆或技能
- 生成媒体或导出文档

你不需要手动创建检查点。

## 限制

Rewind 主要恢复 DesireCore 管理的本地状态。以下外部副作用通常不会被撤销：

- 已发送的邮件或已调用的外部 API
- 已推送到远程仓库的变更
- 通过命令行在 DesireCore 管理范围外手动修改的文件
- 第三方服务内部已经完成的交易或审批

如果回撤涉及外部副作用，建议查看审计记录后再决定下一步处理方式。
